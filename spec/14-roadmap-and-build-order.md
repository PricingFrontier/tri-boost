## 14 — Roadmap & build order

This section sequences the implementation. It owns no types; it orders the decisions the other thirteen sections own. The governing principle is **prove the thesis before broadening it**: the load-bearing claim is *competitive predictiveness with exact ≤3rd-order decomposition*, so the first thing built end-to-end is the thinnest path that demonstrates it — a vertical spine that bins, boosts, accumulates, purifies, and passes the five invariant gates — and everything else is layered on a green spine. We build for I1/I2 from line one: the firewall (`ExactnessMode`, §2.6/§3) and the five `Invariant` checks (§3) are wired in v1, not retrofitted, so no later phase can silently erode exactness.

### 14.1 Three releases, defined by capability not date

- **v1 — GREEN-SPINE ALPHA (the exact-decomposition spine).** The smallest system that fits a fully-decomposable model and proves it *losslessly* — the spine's reason to exist, **not** a benchmark gate. Squared-error first (so reconstruction is testable before any nonlinear link), then the full v1 loss set; the oblivious Newton histogram engine; the heredity/FAST/Sobol interaction funnel with the order-3 budget; accumulate→purify→tables; the five gates; serialization; the Python wheel. The v1 exit gate **is** GREEN-SPINE ALPHA: depth-3 oblivious Newton histogram boosting on `SquaredError` binned, boosted, accumulated, purified, and the **five lossless invariant gates green** (Reconstruction / MassConservation / Purity / VarianceSum / ThreeWayEqual) on every fitted model, with *every* model `Exact`. This proves exact ≤3rd-order decomposition end-to-end; predictiveness ranking is *not* claimed here. The EBM/TabArena benchmark is **deferred to the v1.5 release gate** — broadening is only safe once the spine is green.
- **v1.5 — close the remaining gap (the EBM/TabArena RELEASE GATE).** The predictiveness levers that need the spine underneath them: Fisher sorted-ordinal target statistics + empirical-Bayes shrinkage, multi-distinct-categorical-axis trees, quantized integer histograms (promoted from v2 as the bit-reproducibility mechanism), MVS sampling, CatBoost teacher-distillation, and bagged greedy ensemble selection (default off). This is the bulk of "chase the black boxes." **The v1.5 release gate is the core predictiveness milestone:** with these accuracy levers landed, on TabArena beat EBM/GA2M and land within striking distance of the unconstrained GBMs (XGBoost/LightGBM/CatBoost), with *every* fitted model `Exact`. This is items 1–5 of the gap-closing stack (§02-gap) — "beat EBM, near-parity on most data."
- **v2 — the high-cost, benchmark-gated sharpeners.** Fully-corrective leaf refit, Nesterov/accelerated boosting, DART/`random_strength`, the gradient-based triple detector front-end, and the `joint`/Hooker reference measure as a first-class export. Each is exactness-preserving and each shrinks tree count or variance; none is milestone-critical, and each defaults off pending a pricing-data benchmark.

Cross-phase, non-negotiable: every release must compile on stable, pass `cargo clippy -- -D warnings`, `cargo fmt --check`, `cargo deny`, MSRV 1.64, and the bit-reproducibility gate (train at `n_threads ∈ {1,2,8}`, assert byte-equal serialized model). A phase is not "done" until its gate is green in CI.

### 14.2 v1 — the vertical spine, phase by phase

The spine is built bottom-up so each phase has a testable artifact and the *next* phase consumes only green code below it.

**Phase 0 — scaffold & contract (§02, §13, parts of §10).** Workspace (`pattern-boost-core`, `pattern-boost-py`, `python/`, `benches/`), `#![forbid(unsafe_code)]`, `#![deny(missing_docs)]`, the lint gates, the `Backend` trait seam, and the shared types of §2 as compiling stubs: `PbError` + `Invariant`, `FeatureId`/`AxisProvenance`, `GradHess`, `ObliviousTree`/`Split`, `Model`, `Booster`/`FitSpec`, `ExactnessMode`. No algorithms yet — the goal is that `cargo test` runs and the type contract is frozen. *Gate G0:* the five `Invariant` checks exist as functions returning `Result<(), PbError>` (initially over hand-built fixtures) and the CI matrix (lints/fmt/deny/MSRV) is green.

**Phase 1 — binning (§03).** Build `BinnedMatrix`/`BorderGrid` for real: global per-feature quantile grid from a seeded ~200k subsample, midpoint borders for low-cardinality numerics, the reserved missing bin (bin 0), the merged/union-grid rule, and exposure-offset plumbing (`offset = log(e)`). The learned default direction is decided here but exercised in Phase 2. Output: a deterministic `f32→u8` column-major matrix.

```rust
pub fn bin_columns(x: &[&[f32]], cfg: &BinConfig, seed: u64) -> Result<BinnedMatrix, PbError>;
```

*Gate G1:* binning is bit-reproducible across thread counts (quantile subsample is seeded and order-stable); round-tripping a value through `BorderGrid` lands in the documented bin including the missing bin.

**Phase 2 — the oblivious Newton histogram engine on squared error (§06, consuming §05's `SquaredError`).** The heart of the spine. Per level, build the per-feature `≤8-leaf × bins` gradient/hessian histogram with the subtraction trick (child = parent − sibling), score each `(axis, bin_le)` by the **summed Newton gain** across all `2^level` current leaves

`gain = ½ Σ_leaf [ G_L²/(H_L+λ) + G_R²/(H_R+λ) − G²/(H+λ) ]`,

pick one shared split per level (1..=3 levels, graceful early-termination legal), and fill leaves with the exact weight `w* = −G/(H+λ)`. The `ObliviousTree` constructor enforces I1 (distinct-raw-feature count == depth). Boosting loop: `boost_from_average` intercept `f0 = link(weighted mean)`, the LR×n_trees schedule, deviance early stopping, L2 `λ`, `min_sum_hessian_in_leaf`, `path_smooth`. Full-precision `GradHess` only in v1; quantized histograms are v1.5.

```rust
fn find_oblivious_tree(hist: &HistEngine, gh: &GradHess, cfg: &EngineConfig)
    -> Result<ObliviousTree, PbError>;
```

*Gate G2:* on a synthetic order-≤3 piecewise-constant target the engine recovers it to float tolerance; Newton gain and `w*` match the closed form in unit tests; the model is bit-reproducible across `n_threads ∈ {1,2,8}` (per-thread padded histograms, fixed-order reduce — §11).

**Phase 3 — accumulate → purify → tables (§08).** Walk the ensemble's 8-cell tables onto the merged grid, accumulate per realized `FeatureSet`, then **purify** highest-order-first (3→2→1→intercept, Lengerich mass-moving) under the default `RefMeasure::ProductMarginals { laplace }`, yielding the `TableBank`. Compute Sobol importances `σ²(f_u)/σ²(F)`. Wire all five `Invariant` checks over the *real* bank: Reconstruction (one interior point per merged-grid cell), MassConservation, Purity (per-slice `w`-weighted mean zero), VarianceSum, ThreeWayEqual (tree-sum = table-sum = exact Faith-Shap ≤order-3, agreeing to the derived float tolerance `recon_tol` — §08/§13, not "bit-equal").

*Gate G3 — the lossless gate, the spine's reason to exist:* all five checks pass as build-blocking assertions on every fitted model in the test corpus. proptest drives random order-≤3 ensembles through accumulate→purify and asserts the purification identities. **This is the green spine.** Once G3 holds, the thesis mechanism is proven and broadening is safe.

**Phase 4 — the v1 loss set (§05).** With the squared-error spine green, add `Logistic`, `Poisson`, `Gamma`, `Tweedie { rho }` to the `Loss` trait — exact `(g,h)`, deviance early-stop metric (never RMSE on Poisson/Gamma/Tweedie), `pred_from_raw` via `exp(k·F)` not `powf`, hessian floor `ε`, `max_delta_step` cap, multi-step Newton leaves + Armijo for the non-quadratic losses. Losses are orthogonal to tree shape, so I1/I2 are untouched; the firewall stays `Exact` because the log link is affine-in-score.

*Gate G4:* each loss's `(g,h)` matches the closed form; on a log-link Poisson fixture the exposure offset anchors the base level to 1.000; G3's five checks still pass on every link.

**Phase 5 — interaction selection & constraints (§07).** The accuracy heart: the **heredity + FAST-RSS + Sobol admission funnel** as a *soft prior* (never a hard gate — FAST's RSS ≠ Newton gain, so it may not amputate a real interaction), `max_interaction_order ∈ {1,2,3}` (default 3) as a whole-tree constraint, joint boosting over admitted supports with a single final purification (reject EBM-style mains-then-interactions staging), `MonotoneMap` (name→sign, never positional) via per-level joint leaf-clamp with graceful early-termination, and credibility floors.

*Gate G5:* monotone constraints provably hold on the **total score** and on the **constrained 1D main-effect table** only — a purified 2D/3D interaction slice need NOT be monotone in isolation (per §07's caveat), so the gate checks the total and the 1D table, not every exported cell; `max_interaction_order = 1` yields a pure-additive model (no realized 2D/3D tables); the funnel is verified soft (a planted interaction below the FAST threshold is still recoverable). G3 invariants hold throughout.

**Phase 6 — serialization & the standalone Rust surface (§10, completing §06's `Booster`).** `serde_json` (canonical, self-describing) + `bincode` 2.x (fast binary, via `bincode::serde::encode_to_vec`/`decode_from_slice` under a frozen `bincode::config::standard()` — the pinned config is what makes the binary round-trip byte-equal), `schema_version` round-trip, branch-free 8-cell lookup + table-sum scoring (the hot scoring path is panic-free per §10's policy — `slice.get(..).ok_or(PbError::Internal)?` or a scoped `#[allow(clippy::indexing_slicing)]` fn with a `// JUSTIFIED:` bounds proof + boundary test), `TableBank` JSON export. Never pickle. *Gate G6:* a model and its `TableBank` round-trip byte-stably; scoring via tree-sum equals scoring via table-sum (a runtime echo of ThreeWayEqual).

**Phase 7 — Python bindings (§12).** PyO3 `_pattern_boost`, maturin, **abi3-py310**, sklearn-compatible `PatternBoostRegressor`/`Classifier`, zero-copy `PyReadonlyArray2<f32>` + `into_pyarray`, `py.detach` GIL release with a scoped rayon pool (`n_jobs`), `.pyi` stubs + `py.typed`, `PbError`→exception mapping. The Rust core carries **zero** pyo3 dependency. *Gate G7:* a wheel builds on manylinux2014; a Python-fit model and a Rust-fit model with identical config/seed/data are byte-identical (the firewall extends across the FFI boundary).

**v1 EXIT GATE — GREEN-SPINE ALPHA (the exact-decomposition spine).** All of G0–G7 green, which *is* the green-spine milestone: depth-3 oblivious Newton histogram boosting on `SquaredError` (then the full v1 loss set) binned, boosted, accumulated, purified into the `TableBank`, with the **five lossless invariant gates green** (G3) as build-blocking assertions on every fitted model, every model `Exact`, byte-reproducible across `{1,2,8}` threads and across the Python FFI. This proves exact ≤3rd-order decomposition end-to-end. **It is NOT a benchmark gate** — no predictiveness ranking is claimed at v1. The EBM/GA2M-beating TabArena milestone is the **v1.5 release gate** (§14.3), reached only after the v1.5 accuracy levers land.

### 14.3 v1.5 — closing the gap (the EBM/TabArena RELEASE GATE)

Each lever sits on the green spine, touches a single owning section, and must re-pass G3's five checks before merge. The **v1.5 release gate is the core predictiveness milestone**, deferred here from v1 by design: only once Fisher sorted-ordinal TS, quantized histograms, MVS, distillation, and ensemble selection have landed do we run the development-yardstick benchmark — on TabArena, beat EBM/GA2M and land within striking distance of unconstrained XGBoost/LightGBM/CatBoost, with *every* fitted model `Exact`. (The benchmark is our internal yardstick; we ship no benchmarking tooling.) Until then the spine is green (v1) but unranked.

- **Quantized integer histograms (§06).** `QuantGradHess` with stochastic rounding; integer sums are associative ⇒ order-independent ⇒ this *is* the bit-reproducibility mechanism (and ~2× speed). **Leaf refit from full precision is mandatory** on the log-link path. *Gate:* quantized and full-precision spines agree within tolerance after refit; the `{1,2,8}`-thread byte-equality gate now runs on the quantized path.
- **Fisher sorted-ordinal target statistics + empirical-Bayes shrinkage (§04).** Leakage-free ordered/cross-fitted TS produces a data-driven *ordering*; the split is a sorted-ordinal split so each category stays a distinct table row. Auto-shrinkage `(n_c·mean_c + m·global)/(n_c+m)`. **Combination CTRs forbidden** (>3 raw features → breaks I1). *Gate:* the encoded axis carries correct raw-feature provenance; I1 is enforced on distinct *raw* features; the firewall stays `Exact` (sorted-ordinal, not a continuous TS axis).
- **Multi-distinct-categorical-axis trees (§04/§06).** Up to 3 distinct categorical TS axes per tree → exact cat×cat[×cat] tables. *Gate:* provenance-based budget check passes; G3 holds with categorical axes.
- **MVS sampling (§06).** `p_i = min(1, √(g_i² + λh_i²)/μ)` (the hessian term is `λh_i²`). Seeded, reproducible. *Gate:* determinism gate holds under sampling.
- **CatBoost teacher-distillation training mode (§09).** Default teacher = CatBoost (itself oblivious). Two params: *whether* to distil = `FitSpec.distill: Option<DistillSpec>` (off by default; the teacher is fit only when on), and *how much* = `blend` = true-label weight (default 0.5; `g = blend·g_true + (1−blend)·g_soft`, so `blend=1.0` is the degenerate zero-teacher fit reproducing the base loss, and the soft gradient is just the base `grad_hess` called with the teacher output as target) — touches only the loss *target*, never tree shape, so I1/I2 untouched. Firewall-gated: the distilled model is bit-exact to *its own* tables; only fidelity-to-teacher is <1. *Gate:* a distilled model passes all five checks and exports `Exact` tables.
- **Bagged greedy ensemble selection (§09), default off.** HP-diverse library → union/merged grid → greedy-with-replacement selection on held-out deviance, seeded from top-2, selection bagged → average table banks under a **single shared `w`** in raw score space (`Σα=1, α≥0`) → re-purify once → Sobol-prune for display. Exact by purification linearity. On-ramp: outer-bag table averaging at one HP setting (~8 bags). *Gate:* `purify(Σαf) = Σα·purify(f)` proptest; the averaged bank passes all five checks.

**v1.5 RELEASE GATE — table-budget stress (§07 admission penalty + §08 budget/fallback).** The accuracy levers above (border-rich Fisher-TS axes near 254 bins, union/merged grids over many bagged HP settings) multiply merged-grid cell counts, so the v1.5 gate adds an **adversarial table-size benchmark** alongside the TabArena yardstick: fixtures combining border-rich axes (axes deliberately near the 254-bin cap) with deep bagging (the ensemble's union grid) drive the merged-grid 2D/3D supports to their worst case. The benchmark asserts (a) per-table cell counts respect §08's `max_table_cells` budget and the total-bank budget, with the §07 high-cardinality admission penalty deprioritizing the most expensive triples and the §08 sparse-tensor storage fallback (exact — same tensor, sparsely stored) or a typed `PbError::TableBudget` engaging when budgets are exceeded; (b) purification time stays within the measured envelope on these worst-case grids; (c) peak memory of the bagged union-grid bank is bounded. *Gate:* the five invariant checks still pass on the sparse-stored hot triples; no budget breach silently inflates memory or wall-clock.

### 14.4 v2 — the benchmark-gated sharpeners

- **Fully-corrective leaf refit (§09).** Structure frozen ⇒ linear in leaf values ⇒ ridge/IRLS over `#trees×8`. Fewer trees ⇒ smaller tables. Exactness-preserving; refit then purify.
- **Nesterov/accelerated boosting (§09).** Linear momentum mix of ensembles ⇒ still a sum of ≤3-feature tables; O(1/m²) ⇒ far fewer trees, directly attacking depth-3 weakness.
- **Gradient-based triple detector (§07).** NID/PID-style or META-ANOVA `I(j)=E[Var{∂_j f|X_j}]` on a quick dense reference, seeding the heredity allow-list. Pre-training analysis only — exactness-neutral, never enters the model.
- **Joint/Hooker reference measure as first-class export (§08).** Recompute the bank under `RefMeasure::Joint` *without retraining* (exactness-preserving); importances → Shapley-effects. The standing between-measure relativity-drift diagnostic lives here.
- **Optional knobs (§09):** DART, `random_strength` — exactness-preserving, situational, default off.

### 14.6 Open forks (recommended defaults, carried from the design log)

1. **Fully-corrective refit / Nesterov on by default?** → **Off, benchmark-gated** (v2). Both are exact and both shrink tree count, but per-round cost must be measured on pricing-shaped data before either is defaulted. *This is the one genuinely-open fork; everything else below is resolved.*
2. **Reference measure default** → resolved: `ProductMarginals { laplace }` (v1); `Joint`/Hooker built and benchmarked as a post-hoc export (v2), reversible without retraining.
3. **Quantized histograms placement** → resolved: **v1.5, promoted** as the reproducibility mechanism, refit-leaves-from-full-precision mandatory on log-link.
4. **`max_interaction_order` default** → resolved: **3** (the differentiator); 1/2 exposed as filing strategies, never as predictiveness levers.
5. **Boost-on-GLM-base-margin (CANN)** → deferred past v2; only an affine base-margin stays `Exact`, a >3-order GLM margin flips to `Approximate` via the firewall.

### 14.7 The six architecture-exploit additions — phase slotting

The architecture-exploits investigation surfaced six exactness- and determinism-safe speed wins (the design doc owns the rationale). None is milestone-critical; each slots into an existing phase rather than opening a new one, and none changes a gate's pass/fail condition. They split cleanly into two homes:

- **Inference / perf-hardening phase (rides with P6 scoring + the §11 cross-cutting discipline, hardened in v1.5).** The **ScoringBank / PackedTree** — a load-derived, scoring-only path-A view, `#[repr(C)] { feat: [u8;3], thresh: [u8;3], miss: u8, leaf: [f32;8] }` padded to one 64-byte cache line, with `ScoringBank(Vec<PackedTree>)` contiguous in stored tree order, built once in finalize/deserialize as a byte-exact re-encoding of the same f32 leaves and same compares (the serialized `Model` — the determinism gate's subject — is untouched; the u8 axis caps the hot path at ≤255 features, wider models take a side-table fallback); its row-blocked streaming kernel is the bandwidth multiplier on that bank. The **leaf-select instruction** committed to an in-register permute (AVX2 `vpermps` / AVX-512 `vpermi2ps` / NEON `vqtbl`) over the 8-entry register-resident leaf LUT — *not* a hardware gather (`vgatherdps`, microcoded and load-port-bound) — via the existing `multiversion`/`pulp`/`wide` safe wrappers. The two incremental notes (`predict_binned` + digitize-once amortization; the flat single-arena precomputed-offset path-B kernel) ride here as §10 API/implementation notes, covered by the existing path-equality echo of ThreeWayEqual. *Gate impact: none new* — the byte-stable round-trip and tree-sum == table-sum gate (G6) already audit these as runtime-derived views.
- **Interaction-selection + explainability phase (rides with P5/§07 and P3/§08).** The **wht8** screen — a frozen O(8) Walsh-Hadamard / Mobius transform mapping a depth-3 tree's 8 leaf values to its 8 fANOVA coefficients (1 constant + 3 main `c_i` + 3 pairwise `c_ij` + 1 triple `c_123`) under that tree's per-cut `w`-marginals (exact, O(8); registered under §07, invoked at §06 leaf estimation) — feeding §07's running per-support per-order variance accumulator as a soft online interaction prior, with `c_123²` as a soft online triple-detection witness. **The critical caveat:** the per-tree coefficients live on each tree's *own* 2-point grid under that tree's `w`-marginals; trees cut different borders, so you CANNOT sum coefficients across trees (that drops cross-tree covariance). The wht8-derived per-order variance is therefore a SCREENING SIGNAL, NOT the audited ensemble Sobol — it must NEVER touch the §08 invariant gates (ThreeWayEqual / VarianceSum / Purity / Reconstruction), which stay on the merged-grid purified bank; under `Joint` `w` the clean product form degrades to a heuristic. It is a soft prior that never hard-gates, so it is exactness- and determinism-neutral by construction. (The seductive "wht8 replaces §08 Lengerich purification" idea is WRONG and explicitly rejected — not added.) The **Faith-Shap cross-check** rides alongside: the same wht8 coefficients give a free, independent per-tree order-3 Faith-Shap that must agree bit-close with the §08 mass-moving result — a §13 cross-check oracle (a new invariant test, not a perf claim). *Gate impact: none weakened* — the §08 five gates are unchanged; the Faith-Shap oracle adds one test.

### 14.8 Non-goals (frozen, will not enter any phase)

Interactions beyond 3rd order (the honest bias ceiling — no ensembling crosses it); GPU training (a `Backend` seam is kept GPU-shaped, but no GPU phase is scheduled); ordered boosting (separable, ~1.7× cost, small-data only — research, not a phase); non-symmetric / leaf-wise growth, linear/soft/neural leaves, and combination CTRs (each breaks I1 or I2 and is forbidden on the exact path); any benchmarking, calibration, fairness, uncertainty, or deployment tooling (downstream of the library, out of scope). Each non-goal serves the three aims by *protecting* them: nothing that bends I1/I2 reaches the default path, so MAXIMUM PREDICTIVENESS is pursued only through the exactness-preserving levers above, FULLY DECOMPOSABLE is guaranteed by construction at every phase, and FAST is delivered by the histogram engine, quantized integers, and branch-free table-sum scoring layered onto a spine that was green before any of them were added.

**External tree-compilers are MOOT for pattern-boost.** Treelite, QuickScorer, and lleaves exist to lower a generic tree ensemble into a fast branch-free scorer; the depth-3 oblivious tree is *already their output form* — 3 compares → 3-bit index → 8-entry LUT — so a general compiler buys nothing here. The ScoringBank / PackedTree path-A view (§14.7) and the in-register permute *are* the hand-fit specialization a compiler would emit, made exact and determinism-safe. **GPU inference is roadmap** (a future scoring-path lift on the same path-A/path-B kernels); **GPU training stays an explicit non-goal**, as above. (SQL/LUT deployment is already the `RatingExport`/`RatingTable` artifact of §10 — only a "score in SQL" docs recipe is outstanding, not a phase.)

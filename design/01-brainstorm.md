# tri-boost — Design Brainstorm

> 2026-06-21. The first design brainstorm for the library, synthesizing all prior research ([`../AIM.md`](../AIM.md), [`../research/`](../research/)) into the components, techniques, and approaches for the dual aim: **maximum predictiveness + perfect explainability** — a GBM that predicts as well as the unconstrained incumbents and **beats EBM**, while staying losslessly decomposable into main-effect / 2-way / 3-way tables. **tri-boost is a GBM library, not an insurance tool**: the immediate goal is the *core* (the milestone in §8 — beat EBM on TabArena while exactly decomposable), and insurance rating is the motivating **downstream** application, not the library's scope. *(§§6–7 were cut back to mark that boundary.)*
>
> Produced by a 5-lens design panel — Predictiveness Maximalist, Explainability Purist, Insurance Pricing Actuary, Core Algorithm Architect, Contrarian / Risk Auditor — then stress-tested by a tension auditor and a completeness critic, then synthesized. The individual lenses and the stress-test are preserved under [`brainstorm-panel/`](brainstorm-panel/).
>
> **This is brainstorming — opinionated proposals and recommended defaults, not committed decisions.** Each component is tagged **[EXPLAINABILITY]**, **[PREDICTIVENESS]**, or **[BOTH]**.

---

## The core bet (design philosophy)

The depth-3 oblivious cage is **not a tax on a generic GBM — it is the data structure.** A depth-3 oblivious tree is a rank-1 tensor on a shared grid; the ensemble is a sum of such tensors; explanation is one linear operator (purification); inference is a sum of lookup reads. There is no "model" and a separate "explanation" — there is **one tensor bank in two views** (raw, for the bit-exactness check; purified, for humans and rating), with a build-gated proof they are equal. Everything orthogonal to tree shape — losses, sampling, constraints, calibration, uncertainty — composes natively *because it never touches tree shape*. The whole game, then, is **spending the scarce 3-feature budget deliberately** (so each weak tree lands on real structure) and **defending the exactness boundary obsessively** (so no calibration warp, learned-encoding axis, or linear leaf silently turns "the tables ARE the model" into "the tables approximately resemble the model").

This is the right bet for **tabular data generally**, not a compromise: real tabular signal is overwhelmingly main-effect + low-order interaction — the same bet EBM makes and wins, except we extend it to **exact 3rd order** and refuse to give up accuracy to get there. The proving ground is therefore **TabArena, vs EBM and the unconstrained incumbents** (§8): match the black boxes while staying exactly decomposable and the thesis holds. **Insurance rating is the natural first downstream application** — the tables map directly onto a rating table (log-link → multiplicative relativities, base level = 1.000), and regulators *forbid* the high-order interactions a black box finds — but that is a *consumer* of the library, not its scope. Two artifacts run side by side throughout: the **complete realized support** (losslessness, non-negotiable) for inference, and a **pruned top-k-by-variance** view for human display.

## Components by pipeline stage

### 1. Data, features & representation

- **One shared global binning grid per feature** [BOTH] — the keystone. Quantile borders on a seeded ~200k subsample (v1), computed once, persisted in the model, reused identically at validate/predict/score time. Because all trees share axes, every tree's tensor is exactly representable on the merged grid → accumulation is lossless → purify-then-sum ≡ sum-then-purify (Lengerich linearity). Without one global grid, none of the exactness machinery works. *Decision: the union of realized thresholds per feature is each table's axis (research/03 §4.2) — minimal exact grid.*
- **Midpoint borders for low-cardinality numerics** [BOTH] (v1) — exact splits and one table row per real value for ordinal rating factors (age, NCD years, vehicle group), so tables read 1:1 with reality. Raise `max_bin` on monotone-constrained features so coarse borders don't wipe out all monotone-feasible splits.
- **Categoricals — the Fisher carve-out, resolving the sharpest blueprint conflict** [PREDICTIVENESS, axis-preserving] — Ordered/cross-fitted **Target Statistics** is CatBoost's biggest categorical lever but is `changes_table_form` (the axis becomes a learned encoding). The readability cost is not intrinsic to leakage-free *encoding* — it is intrinsic to *consuming the TS as a continuous axis*. So: compute the leakage-free TS to get a data-driven **ordering**, then apply the **Sorted-by-encoded-mean ordinal split (Fisher, v1.5, `preserves_exactness`)** so each category stays a distinct, human-readable row ("postcode group G → 1.18×") whose order is data-driven. Mandatory **empirical-Bayes auto-shrinkage** (v1, `native`) toward the exposure-weighted base rate so singletons collapse to base. One-hot only for genuinely low-cardinality factors. *This dominates both BP1's raw continuous-TS axis (under-pays on readability) and BP5's blanket ban (over-pays on accuracy).* For regression, raise `ctr_target_border_count` above the default 1.
- **Multi-categorical-axis trees — recover CatBoost's combination edge, safely** [PREDICTIVENESS] (v1.5) — let a depth-3 tree split on **up to 3 distinct categorical TS axes**, giving exact `cat × cat [× cat]` interaction tables (≈1.86% of CatBoost's logloss edge comes from categorical combinations). **Forbid CatBoost-style combination CTRs** — they pack >3 raw features into one encoded axis, so a tree using that axis plus two others touches >3 features and breaks I2. Track raw-feature provenance per axis so the ≤3-feature invariant is enforced on *distinct underlying features*, not encoded columns.
- **Missing values as a designed subsystem** [BOTH] — reserved missing bin + **learned default split direction** (XGBoost sparsity-aware); **collapse-to-base for new/unseen categorical levels** at score time (critical for postcodes); **rare-level collapsing** (`min_data_per_group`). For a *filed* table, "what happens to a policy with a missing/never-seen value" is an audit question — the binning grid + missing handling persist with the model so it is reproducible.
- **Exposure/offset as a first-class argument** [BOTH] (v1 must-have) — `exposure=` sets per-row `offset = log(e)` added to the raw score before the inverse link each iteration; init `F₀ = log(Σwy / Σwe)`. This is what makes `e⁰ = 1.000` the base level and anchors the relativity reading. Same plumbing also enables boost-on-top-of-GLM (§6).

### 2. The learner — depth-3 oblivious boosting engine

- **Oblivious level-wise summed-gain split-finder** [BOTH] — at each of 3 levels, evaluate every `(feature, border)` by applying it to all current leaves simultaneously and summing the **NewtonL2 gain** `½ Σ_leaves[G_L²/(H_L+λ) + G_R²/(H_R+λ) − G²/(H+λ)]`; keep the single argmax for the whole level. One shared split/level ⇒ ≤3 features/tree ⇒ exact 3rd-order fANOVA (I1, I2 by construction). *Decision: Newton gain + exact leaf weights `w* = −G/(H+λ)`, not CatBoost's Cosine default (Cosine forfeits exact leaf weights).*
- **Histogram engine** [PREDICTIVENESS/speed; BOTH/reproducibility] — pre-bin to `u8`, one tiny `≤8-leaf × features × bins` tensor per level, subtraction trick (child = parent − sibling). **Quantized integer g/h histograms with stochastic rounding, leaves refit from full-precision g/h.** Integer sums are *associative* → order-independent → **bit-reproducible across thread counts**, which directly serves the "tables == ensemble" invariant *and* regulated reproducibility — the speed lever (~2×) and the exactness lever are the *same* mechanism. Per-thread private cache-line-padded histograms, fixed-order reduce, no mutex. *Flag: the catalogue scores quantized histograms at v2; promoting to v1.5 as the reproducibility mechanism is a defensible re-prioritization — refit-leaves-from-full-precision is mandatory on the Poisson/Gamma/Tweedie path (the paper found it crucial for regression).*
- **Interaction selection — the accuracy heart** [BOTH] — a soft-prior funnel, never a hard gate, bounded by composition (all five blueprints converge here; adopt verbatim): (1) **heredity/strong-hierarchy** at train time — admit a triple only if its sub-pairs are admitted, a pair only if both mains are — avoids the `C(n,3)` scan (n=100 → 161,700 triples) by *composition not enumeration*; (2) **FAST RSS pre-filter** on the residual using the binned cumulative histograms the engine already has (O(b²)/pair), feeding the interaction-constraint allow-list as a *soft prior*; (3) **exact post-purification Sobol/Shapley-effect variance** as the *final* arbiter. *Critical decisions: FAST's RSS objective ≠ Newton gain, so FAST never hard-gates (it would amputate a real interaction). And reject EBM-style mains-then-interactions staging — it mis-converges under correlation (the GAMI-Tree lesson). **Joint boost over admitted supports, single final purification pass.***
- **Gradient-based triple detector as the funnel's front-end (sharper than FAST for 3-way)** [PREDICTIVENESS] (v1.5/research) — FAST is a 2-way RSS screen, but *triples* are where the order-3 budget actually beats EBM. Seed the heredity allow-list with a feature-interaction-detection statistic (NID/PID-style gradient·Hessian interaction scores, or META-ANOVA's `I(j) = E[Var{∂_j f | X_j}]`) computed once on a quickly-trained dense reference. It is *pre-training analysis* over which triples to admit — exactness-neutral (never enters the model), a pure soft prior. The SIAN/NODE-GA3M *models* are soft/neural (break exactness, don't port); only their *detector* transfers.
- **Leaf estimation** [PREDICTIVENESS] — `leaf_estimation_iterations > 1` (multi-step Newton): re-derive (g,h) at updated leaves each step, sharpening leaves on the non-quadratic pricing losses (Gamma/Tweedie) that are the whole point. Touches only the 8 leaf values of a fixed structure ⇒ exactness untouched. Pair with **backtracking (Armijo)** for ill-conditioned Tweedie. *Tradeoff: cheap on leaf-aggregated sums, expensive for arbitrary-loss per-row optima; default >1 only after benchmarking.*
- **Loss trait + objectives** [BOTH] — `grad_hess` (one pass, w.r.t. raw score F), mandatory `init_score = link(weighted mean)`, `link ∈ {Identity, Log, Logit}`. Ship SquaredError, Logistic, Poisson, Gamma, Tweedie(ρ) in v1; compute log-link powers as `exp(k·F)` not `powf`. Objective is fully orthogonal to tree shape (I1/I2 untouched). `max_delta_step` caps the Newton step for Poisson's exponential instability; hessian floor ε for Logistic/Tweedie corners.

### 3. Predictiveness boosters within the cage

> The highest-leverage additions from the gap-closing analysis ([`02-predictiveness-gap-closing.md`](02-predictiveness-gap-closing.md)) are folded in here — distillation leads, ensembling closes.

- **Teacher-distillation training mode — the highest-upside new lever** [PREDICTIVENESS] (v1.5) — fit a strong teacher (**CatBoost is the natural default**: it is *itself* an oblivious-tree ensemble, so distilling into our depth-3 model just takes the ≤3rd-order projection of the same tree family, and we inherit its best-in-class categorical/ordered-TS signal — which matches our own TS axes), then train the depth-3 oblivious booster against the teacher's **soft score** (or a soft + true-label blend). This changes only the *loss target* — never tree shape — so **I1/I2 are untouched and the student stays exactly decomposable**. The student provably matches the teacher's **≤3rd-order projection**; only genuine ≥4-way teacher structure is irreducible. Evidence (META-ANOVA, arXiv:2408.00973): distillation is near-free, and on some data the additive student *beats* its teacher. Ship behind the exactness firewall — the distilled model is bit-exact to its own tables; only its *fidelity to the teacher* is <1. *(The teacher↔student deviance gap is, incidentally, the cost of the order-3 cap on that dataset — an internal dev read, not shipped tooling.)*
- **boost_from_average → the fANOVA intercept** [BOTH] (v1) — `F₀ = link(exposure-weighted mean)` in score space IS the base rate / `f₀` term. Mandatory, exact, faster convergence. Compute in link space (never `log(mean y)`).
- **Fully-corrective leaf refit** [PREDICTIVENESS] (v2, the one "neural-adjacent" idea that stays bit-exact) — periodically re-optimize *all* trees' leaf values jointly (structure frozen) against the regularized loss. With structure fixed the model is *linear in leaf values* — a Newton/ridge solve over `#trees × 8` params on the already-computed leaf one-hots. Corrects stagewise over-shrinkage, **reduces tree count → smaller tables → double win.** Compose: refit, then purify. *Gate behind a pricing-data benchmark.*
- **Accelerated (Nesterov) boosting** [BOTH] (v2) — O(1/m²) function-space convergence → far fewer trees for the same error, directly attacking "weak-per-tree → need many trees." Momentum combination is a linear mix of ensembles ⇒ still a sum of ≤3-feature tables; fewer trees = more readable tables. *Tradeoff: ~2× per round + earlier overfit (needs early stopping) — benchmark vs lighter Biau AGB first.*
- **MVS sampling over GOSS** [PREDICTIVENESS via budget] (v1.5) — provably dominates GOSS on split-score variance; buys more trees at fixed wall-clock. Validate μ under strongly-varying Tweedie Hessians.
- **Early stopping on deviance, column/row subsampling** [BOTH] (v1) — standard regularization; subsampling also cuts histogram work.
- **Bagged greedy ensemble selection (opt-in) — the sanctioned ensemble path** [PREDICTIVENESS] (v1.5, default OFF) — train K tri-boost models with **diverse hyperparameters**, **greedy-select with replacement on held-out deviance** (Caruana 2004; seed from the top-2 and *bag the selection* so its gain survives out-of-sample), and average the **table banks** on a common merged grid. Exact by purification linearity (`purify(Σαᵢfᵢ) = Σαᵢ purify(fᵢ)`; all members order-3 → the average is order-3); average in **raw score space** under a **single shared `w`** (`Σαᵢ = 1, αᵢ ≥ 0`), re-purify once, Sobol-prune for display. **Variance reduction only — it cannot lift the order-3 *bias* cap.** Inference stays free (one re-purified bank, independent of K); expected ~0.5–1.5% deviance over the best single model. *On-ramp: outer-bag table-averaging at one HP setting (~8 bags) — most of the win + free per-cell error bars, no HP sweep. Honest: a fair "beat EBM" comparison is bagged-vs-bagged (EBM already bags 14× internally).*
- **Optional accuracy knobs (situational, all exactness-preserving)** [PREDICTIVENESS] — DART tree-dropout (non-unit-weight bookkeeping; beats plain MART on some tabular data); function-space line search / 2nd-order step correction (modest, mainly on non-quadratic losses); `extra_trees` / `random_strength` randomized split selection (helps noisy data). Each stays a weighted sum of oblivious trees ⇒ exactly decomposable. Benchmark-gated; none milestone-critical.

### 4. Constraints & structure control

- **Monotone constraints via per-level joint leaf-clamp** [BOTH] (v1) — premium must not fall as risk rises (unfileable otherwise; wrong *sign* costs 2.5–21.2% AUC, right sign <0.2%). CatBoost-style joint pairwise constraint across the 8-cell shared-split table, checked on `w = −G/(H+λ)`. *Decision: `name→sign` dict, never positional. Quirk: if no monotone-valid shared split exists, the level's gain goes to −∞ → **gracefully terminate the tree early at depth <3** (a legitimate lower-order fANOVA outcome, not an error). Document loudly: monotonicity holds on the **total score and the 1-D table, not on each interaction cell in isolation** after purification.*
- **`max_interaction_order ∈ {1,2,3}` as a filing strategy** [BOTH] (v1) — whole-tree interaction-constraint semantics (a tree's ≤3 features must lie jointly within one allowed group, so "permitted interactions" == "tables that can appear"). Order-1 = pure additive GAM (regulator-facing safe harbor); 2 = GA2M (matches EBM); 3 = the differentiator. *This makes tri-boost strictly dominate EBM on its own turf and gives nervous insurers a dial they understand.* Default 3.
- **Credibility/smoothness on the default path** [BOTH] (v1/v1.5) — `min_sum_hessian_in_leaf` as the **exposure/credibility floor** (no rating cell fires on a handful of policy-years); `path_smooth` (parent-shrinkage). *Decision: NO in-training Whittaker-Henderson/ICEnet graduation on the audited path — those break exactness. Default smoothness = credibility floors + bagging; graduation is opt-in post-export with a "not bit-exact" stamp.*

### 5. The explainability engine

- **Accumulate → purify → tables** [EXPLAINABILITY] — expand each tree onto the merged grid, sum into raw tensors `T^raw_u`, then **PURIFY 3-way→2-way→1-way→intercept** (Lengerich Alg. 2, mass-moving). By linearity it streams: maintain a running purified bank, fold in each tree. *State loudly: the decomposition is **complete, not truncated** — ≤3 features means provably no order-≥4 terms, unlike EBM (caps at 2) or post-hoc fANOVA on a black box (truncates and approximates).*
- **Reference measure `w` as a versioned, signed, post-hoc-recomputable input** [EXPLAINABILITY] — *the most consequential call.* Default **Laplace-smoothed empirical product-of-marginals** (respects per-axis density, dodges Hooker extrapolation, stays positive). Offer `uniform` and `joint/Hooker` (hierarchical-orthogonality, faithful under correlation but breaks `σ²(F)=Σσ²(f_u)` and exact equal-split SHAP). **Recomputing tables under a different `w` without retraining is `preserves_exactness`** (leaves stay piecewise-constant, sum conserved) — *the catalogue mis-scores this as breaks/changes-form and it must be corrected.* Stamp `w` on every exported table, importance, and SHAP plot. *Decision: product-`w` for v1 (auditable, sums to 1, SHAP exact, fast); build the joint path early and **benchmark how far relativities move between measures** — that movement is the credibility metric under regulator questioning. Under joint-`w`, switch importances to Shapley-effects (still sum to 1) and label SHAP "interventional."*
- **Importances** [EXPLAINABILITY] — **Sobol `S_u = σ²(f_u)/σ²(F)`** from purified table variances, zero model calls, sums to ~1 under product-`w`. This is the table-selection AND export-ranking primitive, and drives both heredity admission and display pruning.
- **Exact SHAP as O(1) table reads — a provable headline** [BOTH] — `φᵢ(x) = Σ_{u∋i} f_u(x_u)/|u|` (Bordt–von Luxburg equal-split). The stronger result (InstaSHAP / n-Shapley / Möbius): a *purified order-3 additive model simultaneously IS* the fANOVA decomposition, the exact n-Shapley/**Faith-Shap up to order 3**, AND the Möbius transform — so tri-boost claims **exact Faith-Shap interaction indices** (resolving SHAP-interaction non-uniqueness) for free. *Label it "interventional," never just "SHAP." Demote stock TreeSHAP to a test oracle only.* H-statistic and PDP/ICE are thin views over the same tables (`H²_jk = σ²(f_ij)/σ²(PD_jk)`, exact, clamp H>1).
- **Lossless invariants as build gates** [EXPLAINABILITY] — per-cell reconstruction `max_x |F_ens − Σf_u| < tol` (one interior point per merged-grid cell is exhaustive since piecewise-constant); mass conservation; per-slice zero-mean purity; variance-sum identity (branches on `w`); **three-way-equal-number** (tree-sum = table-sum = Shapley-sum, bit-equal). *If these ever disagree, there is no product.* These run as build-blocking assertions, not skippable tests.
- **The exactness firewall** [EXPLAINABILITY] — a typed `Exact`/`Approximate` mode. Anything that breaks the losslessness assertion flips the flag and refuses to export "rating tables" — it exports "rating tables + residual model." This is the structural defense against death-by-a-thousand-cuts (ordered-TS-as-continuous-axis, linear leaves, a calibration spline each "seeming fine" and collectively making the tables a lie).
- **Complete-for-inference / pruned-for-display split** [EXPLAINABILITY] — deployed model uses *all* realized tables; filed view shows top-k by Sobol with honest labelling ("showing 12 of 47 tables, 99.3% of variance"). LUT-sum scoring is lossless only over the *complete* support — never the pruned subset.

### 6. Out of scope — the downstream application layer

tri-boost is **a GBM library, not an all-in-one insurance tool.** Two things here stay in scope because they *are* the model and its explanation: the pricing **objectives** (Poisson / Gamma / Tweedie) are just **loss functions** (the `Loss` trait, §2), and the **decomposition into tables** is the explainability output (§5) — for a log-link model those tables read directly as multiplicative relativities (`f₀` → base rate 1.000, `exp(fᵢ)` → relativities), which is exactly why insurance is the natural downstream fit.

**Everything else an insurer's pricing *function* needs is downstream application work — out of scope for the library core, deferred until the core is proven:**

- **Calibration beyond correctness** — local autocalibration, multicalibration, Venn-ABERS, EDF calibration-band tests. A separate capability with its own libraries.
- **Fairness** — discrimination-free marginalization, subgroup multicalibration, proxy auditing. The transparent tables *enable* an audit for free; performing one is not our job.
- **Uncertainty / distributional output** — conformal / CQR interval wrappers, distributional heads (NGBoost/LSS), per-cell credibility bands, reserving. A different model class and/or a different actuarial application; dedicated libraries exist.
- **Deployment / compilation** — emitting tables as SQL/lookup-join, ONNX, or FPGA/RVV artifacts. We ship the tables in a clean documented format (JSON, never pickle); serving them is MLOps.
- **Workflow orchestration** — frequency-severity (fit-twice-and-multiply), A/E reporting suites, rate-filing tooling. Application glue, not the model.

The one near-the-line item worth remembering: a **single-scalar global mean re-anchor** (fold `δ = log(Σwy / Σwμ̂)` into `f₀` so the model reproduces the dataset total) — cheap, exactness-preserving, and arguably a general GBM nicety rather than insurance-specific. **Optional later add, not core.**

### 8. The core success criterion, validation & the explainability guarantees

- **THE CORE MILESTONE — build this and prove this before anything downstream** [BOTH] — on the **TabArena** benchmark suite, **beat EBM / GA2M** and **come within striking distance of unconstrained XGBoost / LightGBM / CatBoost**, while *every* fitted model stays **exactly decomposable** into ≤3rd-order tables (the lossless invariants below pass). Competitive predictiveness *with* exact decomposition **is** the thesis — everything else waits behind it. EBM is the headline rival (we extend its pairwise cap to exact 3rd order and aim to out-predict it); the unconstrained GBMs are the accuracy ceiling we chase. This is our **development yardstick, measured by us** — not a benchmarking feature we ship.
- **Metrics the model reports about itself** [BOTH] — per-objective **deviance / logloss / RMSE** as the default training & early-stopping metric (deviance is strictly proper for the mean on Poisson/Gamma/Tweedie; RMSE is not). Richer fit diagnostics (lift, Gini, A/E) are a pricing concern → downstream (§6).
- **Lossless invariants as build gates** [EXPLAINABILITY] — the explainability guarantee is *enforced, not asserted*: per-cell reconstruction, mass conservation, per-slice zero-mean purity, the variance-sum identity, and the three-way-equal-number check (tree-sum = table-sum = Shapley-sum, bit-equal). If these ever disagree there is no product; they run as build-blocking assertions. (Engine detail in §5.)
- **Sensible defaults, not a tuning product** [PREDICTIVENESS] — ship strong out-of-the-box hyperparameters (the depth-3 cap shifts the LR × n-trees surface toward more, lower-LR trees) and early-stopping, so a user gets a good model without a search. Exhaustive HPO is the user's to run.

> **In scope vs not.** We *use* TabArena (and EBM as the rival to beat) as our **development yardstick** to prove the core works — that is not the same as *shipping* benchmarking tooling. We do **not** ship benchmarking / model-comparison / "cost-of-the-cap" tooling, we do **not** fit an internal unconstrained reference `F_∞`, and we do **not** ship an order-4+ residual head (it would ride un-tabled structure alongside the tables and break the whole guarantee). Judging a *deployed* model against other models is the user's job. The order-3 cap is an **honest, disclosed limitation**.

## The central tradeoffs and where we sit

| Tradeoff | Position | Why |
|---|---|---|
| **Categoricals** (TS accuracy ↔ raw-level readability) | **center-left, Fisher-sorted** | Recovers ~all accuracy *and* keeps "category → row"; the conflict dissolves because the cost was in *consuming* TS as a continuous axis, not the encoding |
| **3-way budget** (accuracy ↔ table bloat) | **full triple budget, display-pruned** | Decouple inference (complete support, lossless) from filing (top-k by Sobol) — the admit decision and the show decision become independent |
| **Reference measure `w`** (faithfulness ↔ variance-decomposition + exact SHAP) | **product-`w` now, joint-`w` benchmarked** | Product is auditable/sums-to-1/SHAP-exact; recomputable post-hoc without retraining, so the choice is reversible |
| **Leaf form** (smooth-target accuracy ↔ "cell IS relativity") | **hard at constant cells** | Linear leaves break monotonicity *within* a cell (unfileable); credibility floors + bagging + finer borders recover most of the gain |
| **Calibration** (lift/coverage ↔ additivity) | **only affine folds in; rest outside** | Nonlinear warps break `g(Σf_u)=Σg(f_u)`; global re-anchor (exact) is default-on, local autocalibration is an external declared curve |
| **Per-tree strength** (depth-3 weakness ↔ tree count) | **compensate, don't relax** | Accelerated boosting + multi-Newton leaves + fully-corrective refit → fewer, sharper trees → competitive *and* smaller tables |

## The big open design forks (with recommended defaults)

1. **Pricing objectives.** → **Ship Poisson / Gamma / Tweedie as losses (`Loss` trait); defer freq-severity *orchestration* to downstream.** The objectives are in scope (they're loss functions); fit-twice-and-combine is application glue (§6).
2. **Reference measure default.** → **Laplace-smoothed empirical product-of-marginals**, with `joint/Hooker` built early and a between-measure relativity-drift benchmark as a standing credibility check.
3. **Quantized histograms in v1 or v2?** → **v1.5, promoted**, justified as the bit-reproducibility mechanism (not just speed); refit-leaves-from-full-precision mandatory on log-link objectives.
4. **Fully-corrective refit / accelerated boosting on by default?** → **Off by default, benchmark-gated** — both are exact and both shrink tree count, but cost must be measured on pricing data before defaulting.
5. **Boost-on-GLM-base-margin (CANN).** → **Offer, with a hard precondition** (see hedges) — strong adoption story ("keep your filed GLM, we add audited residual cells").
6. **`max_interaction_order` default.** → **3** (the differentiator), with 1/2 exposed as filing strategies.

## The riskiest assumptions & their hedges

- **"Order-3 costs almost nothing on pricing data."** *Stance:* this is a disclosed limitation, not something we instrument. The bet is that ≤3rd-order captures the overwhelming majority of pricing signal; an analyst who needs the exact number compares against their own unconstrained GBM. We do **not** ship an internal `F_∞` or a residual-head escape hatch (it would break the table guarantee). If a book genuinely needs >3-way structure, tri-boost is the wrong tool for that book — and saying so honestly is better than smuggling in un-explainable structure.
- **Silent exactness erosion** (a TS axis, linear leaf, or calibration warp drifting onto the default path). *Hedge:* the typed `Exact`/`Approximate` firewall + build-gated three-way-equal-number invariant. The number deployed and the number audited must be the same number, enforced in CI.
- **Reference-measure instability under correlated features** — with correlated inputs the "pure" effects can be artifacts of empty cells. *Hedge:* default product+Laplace for auditability, but build the joint path early and *measure how far the tables move between measures*; large drift is a credibility problem, so we want to discover it ourselves first. (An internal explainability diagnostic — not a model-comparison benchmark.)
- **Base-margin invariant violation** (BP1's CANN idea). *Hedge:* boosting on a GLM base-margin is only exact **if the GLM backbone is itself ≤3-order additive** — a filed 4-way GLM interaction in the offset silently breaks I2 (`η_GLM + tables` no longer decomposes to ≤3rd order). This must be a *hard precondition checked at fit time*, not a footnote.
- **Depth-3 trees too weak → needing so many trees that speed parity fails.** *Hedge:* accelerated boosting + multi-Newton + fully-corrective refit attack tree count directly, and quantized histograms give ~2× headroom — but the AIM's "honest risk" stands: parity is *to be validated by users against the incumbents they already run*, not assumed (and not something we benchmark in-tool).

# pattern-boost — Predictiveness Maximalist Brainstorm

*Lens: claw back every drop of accuracy lost to the depth-3 oblivious + order-3 cap, without ever breaking exact ≤3rd-order decomposition.*

## 1. The thesis / the bet

The cage (depth-3 oblivious, ≤3 features/tree, additive-across-trees) costs us almost nothing **if we stop spending the interaction budget randomly and start spending it deliberately.** A depth-3 oblivious tree is a weak, blunt instrument — but it is weak in a *known* way (3 shared splits, 8 constant cells), and every weakness has an exactness-preserving counter. My bet: the accuracy gap to unconstrained XGBoost/LightGBM closes from three directions simultaneously — (a) **interaction-aware split selection** so each tree's three slots land on the right three features (FAST/H/heredity guiding the candidate set, not a hard gate); (b) **strong per-tree leaf estimation + acceleration** so each weak tree is squeezed for all it's worth and we need fewer of them (multi-Newton, fully-corrective refit, Nesterov); and (c) **never starting from zero** — boost on a GLM/base-margin so our floor is the incumbent. Categorical accuracy (ordered TS) and adaptive binning are the largest single levers but cost table semantics. Everything here is `preserves_exactness` unless flagged; the only two genuine tradeoffs I'll *pay* are ordered-TS axis semantics and (maybe) the bagging cost.

## 2. Components

### A. Spend the interaction budget optimally (the highest-lift cluster)

- **Joint boosting over realized supports, NOT EBM-style staging** [PREDICTIVENESS]. The GAMI-Tree lesson is decisive: naive two-stage (mains to convergence, then interactions on residual) mis-converges and mis-attributes mass when mains and interactions are correlated. Do a *single joint boost* with order caps, then one purification pass. Mechanism: each round the per-level split search finds the best shared `(feature,threshold)`; allow up to 3 distinct features. Tradeoff: none vs staging — strictly better convergence; the only cost is you can't pre-prune the table set cheaply, which (c) below fixes.
- **Interaction-aware candidate selection via FAST + heredity as a *train-time admission filter*** [BOTH]. This is the net-new idea over the inventory's "explain-time pruner" framing. FAST (4-quadrant RSS-on-residual, O(b²)/pair via cumulative histograms — maps perfectly onto our binned engine) ranks pairs; **heredity/strong-hierarchy** admits a triple only if its sub-pairs cleared the screen. Feed this into the interaction-constraint machinery so a tree may open a 3rd slot on `{i,j,k}` only if `{i,j},{i,k},{j,k}` are admitted — you get the order-3 accuracy lever *without scanning C(n,3)*. Tradeoff: FAST's RSS objective ≠ the booster's Newton summed-gain, so use it as a **soft pre-filter feeding constraints**, with final selection by exact purified Sobol variance — never a hard gate that amputates an interaction the GBM would have found. Predictiveness win: the three slots get spent on real structure; explainability win: bounded, hereditarily-coherent table set.
- **H-statistic from exact tables, free, to re-rank kept interactions** [BOTH]. `H²_jk = σ²(f_ij)/σ²(PD_jk)` is an exact variance ratio with zero model calls (the H numerator IS the centered `f_ij`). Use it to drop near-zero interaction tables after training — recovers accuracy budget (fewer, sharper interactions) and readability together. No tradeoff; only guardrail is H's known pathologies (clamp >1, weak-main blowups) so keep Sobol/Shapley-effects as canonical.

### B. Squeeze each weak tree, and need fewer of them

- **NewtonL2 split gain + multi-step Newton leaf estimation** [PREDICTIVENESS]. Default to `ΣG²/(H+λ)` for split ranking and `w=−G/(H+λ)` for leaves (XGB/LGBM parity; CatBoost's GPU-only restriction is an engineering accident, not math). The real lever is `leaf_estimation_iterations > 1`: re-derive (g,h) at updated leaves each step — sharpens leaves on the **non-quadratic pricing losses (Gamma/Tweedie/MAE) that are our whole point**. Touches only the 8 leaf values of a fixed structure ⇒ exactness untouched. Tradeoff: cost path — cheap if recomputed on leaf-aggregated sums, expensive if arbitrary-loss per-leaf optima need per-row re-eval; default >1 only after benchmarking. Pair with **backtracking (Armijo)** for ill-conditioned Tweedie.
- **Fully-corrective leaf refit — the one "neural-adjacent" idea that stays bit-exact** [PREDICTIVENESS]. Periodically re-optimize *all* trees' leaf values jointly (structure frozen) against the regularized loss. With structure fixed the model is *linear in leaf values* and the design matrix is exactly the per-tree 3-bit leaf one-hot (already computed at inference) — a Newton/ridge IRLS solve over `#trees × 8` params. Corrects stagewise over-shrinkage, **reduces tree count → smaller tables → double win**. Composes with purification (refit, then purify). Tradeoff: per-round global solve cost — gate behind a benchmark on pricing data; this is where GrowNet's "corrective step" lives, captured exactly and losslessly.
- **Accelerated (Nesterov) boosting** [BOTH]. AGBM gives O(1/m²) function-space convergence — order-of-magnitude *fewer trees* for the same error, directly attacking the "weak per tree → need many low-lr trees" risk. Pure boosting-loop modification; momentum combination is a linear mix of ensembles ⇒ still a sum of ≤3-feature tables. Fewer trees = more readable tables. Tradeoff: 2-trees-per-iteration (~2× per round) and earlier overfitting (needs early stopping) — benchmark vs the lighter Biau AGB before committing.

### C. Never lose to the incumbent

- **Boost on a GLM base-margin (CANN-style)** [BOTH]. Set `offset = η_GLM(x)` so trees model only the GLM residual; the floor is the filed GLM. Strong pricing deployment story. Tradeoff (explainability caveat I'll accept): the exact tables then explain the *residual*; the full audit trail is `η_GLM(x) + tables(x)`, **only safe if the GLM backbone is itself ≤3-order additive** (guard against a filed 4-way interaction). The base-margin machinery is the same as the exposure offset (`offset=log e`) — zero extra plumbing.
- **boost_from_average as the fANOVA intercept** [BOTH]. F₀ = link(exposure-weighted mean) in score space — IS the base rate and the `f₀` term. Mandatory, exact, faster convergence. Compute in link space exposure-weighted (never `log(mean y)`); Newton-step for closed-form-less Gamma/Tweedie.

### D. Categorical accuracy — the biggest single lever, with a named cost

- **Ordered Target Statistics + empirical-Bayes auto-shrinkage** [PREDICTIVENESS, changes table form]. CatBoost's documented biggest categorical edge. Encode each categorical to *one ordered numeric axis* (leakage-free via ordering or K-fold cross-fit), with sklearn-style empirical-Bayes auto-`m` shrinkage toward the exposure-weighted base rate — *non-negotiable for insurance* (a singleton postcode must collapse to base or the table is over-confident). **The tradeoff I'll pay:** the axis becomes a learned target-statistic encoding, not raw levels — still exact, still order-3 (a correction to records that called this breaks_exactness), but the "category → relativity" reading is partially eroded. Mitigate with the **sorted-by-encoded-mean ordinal split** (Fisher): order categories by their TS, put a numeric border between them — recovers partition-split *quality* while keeping each category a distinct, ordered, human-readable row. For regression, raise `ctr_target_border_count` well above the default 1 (median binarization throws away continuous-severity signal).
- **The constructive carve-out over CatBoost combinations** [BOTH]. CatBoost's categorical *combinations* pack a multi-feature interaction into one axis → a tree splitting on it + 2 features touches 4 raw features → **breaks the invariant; skip.** Instead, let one oblivious tree split on up to 3 *distinct* ordered-TS axes — cat×cat×cat interactions surface as exact purified 2D/3D tables over raw factors. Strictly better than combinations for our product: real interaction accuracy, no opaque crossed axis, no provenance bookkeeping.

### E. Adaptive binning / table-grid resolution

- **Per-feature adaptive borders + midpoint borders for low-cardinality** [BOTH]. Quantile (equal-count) borders on a ~200k weighted subsample by default; **midpoint borders when distinct-count < max_bins** give exact splits and 1 row per real value for ordinal rating factors (age, NCD years, vehicle group) — table reads 1:1 with reality. Raise `max_bin` on monotone-constrained features (too-coarse borders can wipe out all monotone-feasible splits). Tradeoff: finer bins = more accuracy but less stable/readable relativities — expose per-feature border count. **GreedyLogSum** border objective for accuracy in dense regions; the static-init-Hessian weighted sketch is research-only (gain is small for log-link whose init Hessian is near-constant in F).

### F. Sampling / speed → more trees at fixed budget

- **MVS over GOSS; quantized-gradient training** [PREDICTIVENESS via budget]. MVS (`p_i = min(1, √(g_i²+λh_i²)/μ)`, reweight `1/p_i`) provably dominates GOSS on split-score variance — build MVS, skip GOSS. **Quantized integer-gradient histograms** (stochastic rounding, unbiased; refit leaves from full-precision g) give ~2× end-to-end *and* associative→bit-reproducible sums (serves the lossless invariant and regulated reproducibility). Both buy more trees / faster iteration at fixed wall-clock, which an accelerated weak learner converts directly to accuracy. Tradeoff: MVS μ is sensitive under strongly-varying Tweedie Hessians — validate; turn quant-leaf-refit ON for Poisson/Gamma/Tweedie (paper found it crucial for regression).

### G. Ensembling

- **EBM-style outer-bag table averaging (linear blend only)** [PREDICTIVENESS + uncertainty]. A linear blend of pattern-boost models is itself exact (sum of scaled ≤3-feature tables). Bag N oblivious ensembles, average their tables on a common merged-threshold union grid → smoother tables + per-cell credibility error bars. Tradeoff: ~8–25× training cost threatens the speed bar and the accuracy lift is marginal — **opt-in, default off, sold on credibility not accuracy.** The cheap half (inner row-subsample bags) ships independently. **Hard rule:** any non-linear meta-learner or black-box base flips to breaks_exactness — reject sklearn's default `StackingRegressor` (RF final estimator).

## 3. The 3–5 highest-leverage ideas (my lens)

1. **FAST + heredity as a train-time interaction *admission filter* feeding the constraint engine.** This is the single biggest accuracy-per-unit-explainability-risk lever: it directs the scarce 3-feature budget onto real structure, bounds the table set, and is fully exact. The whole game is spending the cage's budget well; this is how.
2. **Fully-corrective leaf refit.** The only "go-faster/stronger" idea from the soft/neural literature that is 100% exact-safe — sparser ensembles, lower variance, smaller tables, linear-in-leaves solve over already-computed leaf one-hots.
3. **Ordered TS + auto-EB shrinkage + Fisher-sorted ordinal split.** Largest raw accuracy lever on real pricing data (categoricals dominate), and the Fisher-sorted-axis trick recovers most of the readability cost — the best risk-managed version of the one tradeoff worth paying.
4. **Boost on a (≤3-order) GLM base-margin.** Guarantees we never lose to the incumbent; converts the whole project from "beat the GLM" to "improve on the GLM," which is a far safer accuracy floor.
5. **Accelerated boosting + multi-Newton leaves.** Directly neutralizes the "depth-3 trees are weak per tree" risk — fewer, sharper trees → competitive accuracy AND smaller tables.

## 4. What I would explicitly NOT do, and why

- **Linear / piecewise-linear leaves (even ≤3-feature GBDT-PL).** Real accuracy on smooth targets, but it converts constant-cell tables into piecewise-linear shape functions (changes table form) AND monotonicity is unsolved within a cell — a "monotone" model can be non-monotone inside a cell, unacceptable for filed rates. Production LightGBM `linear_tree` is worse — it couples all path features and breaks order-3 outright. Research-only, gated on smooth extrapolation becoming a hard requirement.
- **CatBoost categorical combinations / cross-features / feature hashing.** Combinations pack >3 raw features into one axis → silently breaks the invariant. Hashing merges unrelated categories into one row (unreadable). The distinct-TS-axes carve-out delivers the same interaction accuracy safely.
- **Soft/neural oblivious trees (NODE, GRANDE, TEL, NODE-GAM, GrowNet, DeepGBM).** Every one buys accuracy precisely by going soft/oblique/feature-forwarding/DenseNet-stacked — destroying obliviousness or exploding interaction order far past 3. They define the boundary, not the candidate set. (File the STE / smooth-step *soft-train-hard-infer* primitive as a research seed for a differentiable but axis-aligned oblivious variant — that one could stay exact.)
- **Non-symmetric growth (Depthwise/Lossguide) and `gblinear`.** Violate I1 by definition. Keep them only as honest benchmark baselines to report where the order-3 cap loses.
- **L1/median objective for pure premium.** Targets the conditional median; on right-skewed severity it systematically under-prices. Robust *point/interval* estimation only — use Gamma/Tweedie for the mean.
- **Stacking with a non-linear meta-learner, GBM+LR leaf-embedding into a downstream model, or any post-link calibration folded into leaves.** All reintroduce an opaque, non-additive composite and destroy the "tables ARE the predictor" thesis. Calibration warps live *outside* the table sum as a declared score→prediction map, exactly where the link sits — never re-decomposed.

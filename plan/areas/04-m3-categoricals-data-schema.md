# Milestone 3 — Categoricals, Train/Serve, Model Schema (the data-correctness milestone)

## How this area's structure makes quality automatic

This milestone is where the model first ingests *target-dependent* features, and that is exactly where exactness can silently rot: a leaked target statistic still passes a numeric reconstruction check on the *training* matrix while corrupting the deployed model. The structural defense is **two typed matrices that cannot be confused** (`TrainBinnedMatrix` / `ServeBinnedMatrix`, §03.2a) plus **provenance that counts raw features, never encoded columns** (§2.1/§04.2). Because the foundation areas already wired the five Invariant gates (Reconstruction/MassConservation/Purity/VarianceSum/ThreeWayEqual, §13.1) and the determinism gate (§13.4) into the trunk *before any feature code*, every categorical task here inherits those gates for free — the only new work is (a) routing accumulation through the frozen encoder so the audited bank equals the *served* function, and (b) making the budget gate dereference `provenance[axis].raw`. I sequence so that the **schema/round-trip** lands first (a model can't serve cats it can't persist), then the **encoder math under closed-form tests**, then **leakage avoidance gated by the shuffled-target leakage test**, and only then **multi-cat trees** — so the I1 provenance gate (cat×cat×cat = 3, combination-CTR rejected) and the audit-on-serve gate are written *alongside* the feature, never bolted on. Definition-of-done is always "named gates green," and every task cites its spec §. Gate machinery, lint set, no-panic policy, proptest harness, and the five-check fixture are owned by the foundation/through-line areas and merely *referenced* here.

Prerequisite: the **green spine** (numeric SquaredError, the five gates green, determinism gate green, serde round-trip) is merged. These tasks layer onto it; each must re-pass the five gates (G3) and the determinism gate before merge.

## Sequenced tasks

### M3-T01 — ModelSchema / ObjectiveTag registration + round-trip
**Spec:** §2.6, §03 (R-SCHEMA registration), §05 (`ObjectiveTag`/`LossId`), §10.
**Deliverable:** Add the `ModelSchema { feature_names: Vec<String>, feature_kinds: Vec<AxisKind>, cat_encoders: CatEncoderStore, class_labels: Option<Vec<String>>, objective: ObjectiveTag }` block to `Model` (registered as a §06 Model sub-struct), `ObjectiveTag { link, loss: LossId, tweedie_rho: Option<f32> }`, and an *empty/numeric-only* `CatEncoderStore` stub (no encoders yet). Wire `schema_version` to cover it. Implement `serde::{Serialize,Deserialize}` for all three so they ride the existing serde_json + bincode 2.x (`encode_to_vec`/`decode_from_slice`, frozen `bincode::config::standard()`) paths.
**Dependencies:** green spine (Model + serde).
**Definition of Done:** §13.3 serialization proptest extended — `model == decode(encode(model))` with the schema present, `schema_version` survives, scoring round-tripped model bit-identical; bincode bytes byte-equal across two encodes. `clippy -D warnings` (incl. `deny(missing_docs)`), `fmt --check`, doctests, `cargo deny`, MSRV-1.64 job all green. The five gates + determinism gate still green (numeric-only model, schema is inert). **Size: S.**

### M3-T02 — CatEncoder / CatEncoderStore / TsConfig types + total lookup
**Spec:** §04.2, §04.3 (type block), §2.6.
**Deliverable:** `TsEncodingId` (`#[repr(u8)]`, append-only), `LeakageScheme`, `Smooth`, `TsConfig`, `CatLevel`, `CatEncoder { raw, id, labels, encoding, base, config }`, and `CatEncoderStore { encoders: Vec<CatEncoder> }` with `get(id, raw) -> Result<&CatEncoder, PbError>` returning `PbError::Internal { what }` (never panics) on an absent id. All serde-derive; `CatEncoderStore` replaces the T01 stub. No encoding *logic* yet — just types + total, deterministic lookup.
**Dependencies:** M3-T01.
**Definition of Done:** unit test: `get` on a populated store resolves; absent id → `Err(PbError::Internal{..})`, no panic (the §13.8 correct-variant-per-fallible-fn rule). Serialization proptest now covers a non-empty `CatEncoderStore` (round-trip + byte-stable bincode). All foundation gates green. **Size: S.**

### M3-T03 — Exposure-weighted base rate + `enc(c)` shrinkage math (closed-form gated)
**Spec:** §04.3 (`enc(c) = (n_c·mean_c + m·p)/(n_c+m)`), §03.7 (exposure base rate `p = Σwy/Σwe`).
**Deliverable:** the link-appropriate exposure-weighted base rate `p` and the shrunken `enc(c)` kernel, plus `Smooth::Auto` (`m ≈ within_var/between_var`, weighted/deviance-aware variance form, frozen estimator). Accumulators use the §06 fixed-order reduction discipline (no rayon `reduce`/`sum` in steal order) so encodings are thread-count-independent. Non-finite target into the variance estimate → `PbError::InvalidInput`.
**Dependencies:** M3-T02; consumes §05 `Loss`/`Link` + §03 exposure offset (both on the green spine).
**Definition of Done:** §04.10 math unit tests — `enc(c)` against the closed form; exposure-weighted `p` against hand-computed `Σwy/Σwe`; **shrinkage-correctness** test (singletons collapse to within float tol of `p`; `Smooth::Auto`'s `m` matches closed-form `within_var/between_var` on synthetic data). Reproducibility: encoder fit at `n_threads∈{1,2,8}` produces byte-identical encodings (feeds the §13.4 determinism gate). No-panic lints green. **Size: M.**

### M3-T04 — Leakage-free `fit_cat_encoder` (Ordered + KFold) + leakage gate
**Spec:** §04.3 (Ordered prefix / KFold cross-fit; plain mean **forbidden**), §1/§04.3 (deterministic re-seed `Pcg64::seed_from_u64(splitmix64_mix(seed,round,stage,block))`, `block = FeatureId`), §04.8.
**Deliverable:** `fit_cat_encoder(raw, levels, y, weight, cfg, seed) -> Result<(CatEncoder, Vec<f32>), PbError>` returning the **frozen full-data** `CatEncoder` *and* the per-row **training** (out-of-fold/prefix) encodings. `Ordered{n_perms}` (prefix over a seeded permutation σ; `y_k` never enters its own row) and `KFold{k}` (other-folds means), both drawn from the frozen re-seed — *not* a splittable PRNG. Regression target pre-quantized into `target_borders` (default 16). No runtime dep on sklearn's `TargetEncoder`.
**Dependencies:** M3-T03.
**Definition of Done:** §04.10 **leakage gate [GATE]** — train on a shuffled target, assert encoded-feature Sobol importance does not spike (run for both `Ordered` and `KFold`). **No-self-leak property (proptest):** for `Ordered`, encoding of row `k` invariant to changing `y_k`. **Reproducibility:** byte-identical `CatEncoder` + per-row encodings across `n_threads∈{1,2,8}`; the per-column re-seed asserted position-stable (§13.4). All foundation gates green. **Size: L.**

### M3-T05 — TrainBinnedMatrix / ServeBinnedMatrix construction (the audit seam)
**Spec:** §03.2a (the two role types), §04.3 (audit-on-serve), §03.4 (numeric binning fold-independent → Train==Serve on numerics).
**Deliverable:** the `TrainBinnedMatrix(BinnedMatrix)` / `ServeBinnedMatrix(BinnedMatrix)` newtypes (owned by §03) plus the two constructors: Train uses the leakage-free per-row encodings from M3-T04 for cat axes; Serve re-encodes raw categoricals through the **frozen `CatEncoderStore`** (`Model.schema.cat_encoders`), unseen level → `CatEncoder.base`. Numeric axes share the frozen global grids byte-identically. The cat axis's `AxisProvenance { raw, kind: CategoricalTS{encoding} }` carries the **single** raw `FeatureId`.
**Dependencies:** M3-T04 (encoders), M3-T01 (schema lives on Model).
**Definition of Done:** §04.10 **train/serve consistency** test — round-trip `CatEncoderStore` through serde (§10) and assert `ServeBinnedMatrix` re-encoding equals the frozen full-data map; **unseen level → base**. Property test: numeric columns of Train and Serve are byte-identical (only cat axes differ). No-panic lints + determinism gate green. **Size: M.**

### M3-T06 — Fisher sorted-ordinal split encoding (category stays a row)
**Spec:** §04.4 (sort distinct levels by `enc(c)` ascending → one bin per rank → ordinary `bin<=bin_le`), §04.12 (rare-level collapse runs *before* the sort), §03 (binning of the ordered levels).
**Deliverable:** the *encoding half* of the Fisher split: order levels by `enc(c)`, run §04.12 rare-level collapse first (levels with weighted count `< min_data_per_group`, default 10, → one shared `«rare»` bucket re-encoded by `enc(·)`, member labels retained), then bin the ordered level set (one bin per level for low cardinality, quantile-merge for very high). The fitted ordering + labels are stored in `CatEncoder` so the exporter renders ordered category rows. Ships the **static** TS-sorted axis (per-level `Σg/Σh` re-sort is an explicit §04.11 fork, deferred). The emitted axis is a single piecewise-constant numeric column.
**Dependencies:** M3-T04 (`enc(c)`), M3-T05 (axes feed both matrices).
**Definition of Done:** unit tests: ordering is `enc`-ascending and reproduced at serve; `«rare»` bucket slots into rank order by its own `enc`, one bin per rank (one-bin-per-rank invariant). **§03.10 realized-subset invariant** holds (realized split borders ⊆ persisted `BorderGrid`). The five gates + determinism gate green on a fitted model with one cat axis (exactness preserved — Fisher sorted-ordinal is not a continuous TS axis, model stays `Exact`). **Size: M.**

### M3-T07 — One-hot for low cardinality + dummy reassembly provenance
**Spec:** §04.5 (one-hot for cardinality ≤ `one_hot_max_size`, default 2; border at 0.5), §04.5 rules 1–2 (reassemble dummies onto one axis at table time; all dummies share **one** `AxisProvenance.raw`).
**Deliverable:** the one-hot path for ≤`one_hot_max_size` factors (each level a 0/1 axis, single 0.5 border, leakage-free), with every dummy of a source column carrying the **same** `raw` `FeatureId`. Register the reassembly hook §08 consumes (dummies of one column collapse to one `FeatureSet`), so purification does not fragment the main effect or manufacture spurious within-categorical interactions over the structurally-empty all-ones cell.
**Dependencies:** M3-T05 (matrix construction), M3-T06 (encoder/provenance plumbing).
**Definition of Done:** §04.10 **budget enforcement** sub-test: one-hot dummies of one column count as **one** raw feature in `check_feature_budget`. The five gates green on a one-hot model (exact 1.000-reference row); MassConservation/Purity unaffected by the shared-`raw` reassembly. No-panic lints green. **Size: M.**

### M3-T08 — Multi-distinct-categorical-axis trees + the I1 provenance gate
**Spec:** §04.6 (≤3 distinct categorical axes per tree → exact cat×cat[×cat]), §04.7 (combination CTRs **hard-forbidden** by construction), §2.5/§13.2 (`check_feature_budget` dereferences `provenance[axis].raw`).
**Deliverable:** confirm/exercise that the §06 split-finder admits up to 3 distinct **categorical** axes with no per-iteration overhead (a cat axis is indistinguishable from numeric at split time) and that the `ObliviousTree` constructor's distinct-raw-feature count dereferences provenance. Add the encoder-side structural guarantee that *no* code path emits an axis with more than one `raw` `FeatureId` (combination CTRs unimplemented; no config flag).
**Dependencies:** M3-T06 (cat axes exist), M3-T07 (one-hot provenance).
**Definition of Done:** §13.2 **I1 (feature-budget) gate [GATE]**, categorical arm — proptest over fitted models with cat axes proves a `cat×cat×cat` tree counts as **3** distinct raw features and is admitted; a **4th distinct categorical** is rejected with `PbError::InvariantViolated{FeatureBudget}`; a (synthetically constructed) combination-CTR-style >3-raw axis is rejected at `ObliviousTree` construction with the same variant. §13.8 correct-variant test for the budget breach. **Size: M.**

### M3-T09 — Audit-on-serve: accumulate the TableBank on ServeBinnedMatrix
**Spec:** §04.8 / §08 (I2 between the served function and the bank, both on `ServeBinnedMatrix`; `explain()`/accumulation re-encode through frozen encoders, **never** the noisy `TrainBinnedMatrix`), §13.1.
**Deliverable:** wire the §08 `accumulate`/`explain` entry points (built on the green spine) to take a `ServeBinnedMatrix` for any model with cat axes — the engine builds (or is handed) the Serve matrix by re-encoding raw cats through `Model.schema.cat_encoders` before accumulating. No table accumulation ever touches `TrainBinnedMatrix`. (Implementation of accumulate/purify is the explainability area's; this task is the categorical *routing* + the gate that proves it.)
**Dependencies:** M3-T05 (Serve matrix), M3-T08 (multi-cat trees produce 2D/3D cat tables).
**Definition of Done:** §04.10 **audit-on-serve gate [GATE]** — fixture whose out-of-fold and full-data encodings *differ* on a cat axis; assert the five §13.1 checks pass against the **served** function on the Serve matrix (they would fail if accumulation reused the Train encodings). §04.10 **exactness** test: a fitted model with categorical axes passes all five checks and stays `ExactnessMode::Exact`. ThreeWayEqual (tree-sum = table-sum = Shapley-sum) green with cats present. **Size: M.**

### M3-T10 — Missing-value subsystem completeness + exposure plumbing with categoricals
**Spec:** §03.6 (reserved bin 0, learned `Split.missing_left`, all-missing axis → graceful lower-order termination), §03.7 (exposure offset, `F0 = link(Σwy/Σwe)`), §04.12 (rare-at-train vs unseen-at-serve stay distinct).
**Deliverable:** ensure the missing subsystem and exposure plumbing (largely on the green spine via §03/§06) compose correctly with categorical axes: missing-bin (bin 0) mass carried in `Hist` and routed by learned `missing_left` identically at fit-accumulation and serve-scoring (R-MISSING canonical low-bit); `«rare»`-at-train (own bucket) distinct from unseen-at-serve (→ `base`); exposure base rate folds into the cat encoder's `p` (§04.3). Validate exposure (length, positivity → `PbError::ShapeMismatch`/`InvalidInput`).
**Dependencies:** M3-T05, M3-T09.
**Definition of Done:** §13.5 **degenerate-inputs** test extended with cats: all-missing categorical column, zero-weight rows, zero/negative exposure each yield a valid model or a typed `PbError` — never panic/NaN/Inf (an `is_finite()` sweep over all leaves and table cells). §03.10 exposure test: `F0` matches `link(Σwy/Σwe)`, offset leaves grids byte-identical. The five gates + determinism gate green on a model with cats + missing + exposure together — the full M3 surface exercised end-to-end. **Size: M.**

---

**Ordering rationale.** T01→T02 give a serializable home before any encoder exists (a model that can't persist its cats can't serve them). T03 proves the math against closed form before T04 wraps it in leakage avoidance (gated by the shuffled-target leakage test — the single test that catches the cardinal sin of this milestone). T05 introduces the Train/Serve seam only once both encodings exist. T06/T07 produce the actual axes; T08 turns on multi-cat trees *with* the I1 provenance gate written alongside; T09 closes the audit-on-serve loop (the load-bearing I2 statement for categoricals); T10 verifies missing/exposure compose. Nothing builds on an unverified foundation: every task's DoD is named gates green, and the five Invariant gates + determinism gate (owned by the foundation areas) re-run on every merge, so exactness with categoricals present is a byproduct of the sequence, not an audit step.

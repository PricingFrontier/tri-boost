# Milestone 1 — Green-Spine Alpha (the exact-decomposition vertical slice)

> Spec refs throughout: §00 (canonical types §2, invariant contract §3), §03 (binning), §05 (SquaredError only), §06 (engine), §08 (purify/TableBank/five checks), §10 (scoring), §13 (gate machinery), §14 v1 exit gate (Phases 0–3, 6 of §14.2). This area builds the thinnest path that *proves the thesis*: bin → boost (depth-3 oblivious Newton, SquaredError) → accumulate → purify → TableBank, with **tree-sum == table-sum** and **bit-identical across `{1,2,8}` threads**. Per §14, "broadening is only safe once the spine is green."

## How the structure makes quality automatic

The sequencing is the quality mechanism. **Gates land before features** (M1.0): the lint wall (`-D warnings`, the no-panic set `unwrap_used`/`expect_used`/`panic`/`indexing_slicing`, `#![forbid(unsafe_code)]`, `#![deny(missing_docs)]`, `overflow-checks=true` all profiles, `cargo deny`, MSRV 1.64, doctests — all from §13.8/§13.9) is the *first* PR, so the first line of `bin()` cannot merge if it panics or is unformatted. **The five invariant checks and the determinism gate exist as `Result`-returning functions over hand-built fixtures in M1.0** (Gate G0, §14), then get pointed at real banks the moment a tree exists — so "tables == ensemble" and thread-invariance are build-blocking from the first tree, never an afterthought audit. The §3 firewall (`ExactnessMode`) is wired at the type level, so the spine cannot *not* emit `Exact`. Each task below is a vertical slice that is end-to-end runnable and gated; **Definition of Done is gates-green, not code-written**, and each names the exact §13 gate it must pass. The canonical missing-routing bit (`low = if bin==0 { missing_left } else { bin <= bin_le }`, R-MISSING) is written once and reused identically at split-eval, sample→leaf update, accumulation, and both scorers — this single-source rule is what the ThreeWayEqual gate enforces.

Foundation/cross-cutting areas own the gate *definitions* (§13 lint set, determinism harness, `cargo deny`/MSRV CI jobs, the §11 fixed-`CHUNK_ROWS` reduction discipline). This plan *consumes* them and wires each named gate to the spine's CI jobs.

### M1.4 — Oblivious Newton split-finder + exact leaves + learned missing routing
- **Spec:** §06.2 (summed Newton gain, feature-budget guard I1), §06.4 (`w*=−G/(H+λ)`, leaf storage), §2.5 R-MISSING (canonical low/left bit), §06.6 sample→leaf, §14 Phase 2.
- **Deliverable:** `find_oblivious_tree`/`grow_oblivious_tree` (§14 P2 / §6.6 signatures). Level-wise (depth 1→3) scan: prefix sweep over each `(axis,bin_le)`, `LevelGain = ½Σ_leaf[G_L²/(H_L+λ)+G_R²/(H_R+λ)−G²/(H+λ)]`, pick the single best shared split. Missing evaluated both sides → learned `Split.missing_left`. **The one canonical routing rule** `low = if bin==0 { split.missing_left } else { bin <= bin_le }` written once, used at split-eval AND sample→leaf update. Feature-budget guard on `provenance[axis].raw` (I1). Graceful early-termination (no admissible/`LevelGain≤0`/`<min_split_gain` → `depth<3`, not an error). Exact leaves `w*=−G/(H+λ)·lr` from **full-precision** `GradHess`, `[f32;8]` indexed `b0|b1<<1|b2<<2`, unused tail `0.0`. `ObliviousTree` constructor enforces I1 → `PbError::InvariantViolated{FeatureBudget}`.
- **Dependencies:** M1.3.
- **DoD (gates):** **Newton-gain unit test [GATE] (§13.5/§6.8):** summed gain and `w*=−G/(H+λ)` match by-hand constants exactly on a hand-built 2-leaf histogram. **I1 gate [GATE] (§13.2):** `check_feature_budget` green via proptest over fitted trees — depth∈1..=3, distinct-raw count==depth; early-termination yields valid lower-depth tree, not error; a >3-raw axis rejected at construction. **Equivalence (§6.8):** leaf-refit matches brute-force per-leaf Newton solve. Lint/no-panic gates.
- **Size:** L.

## Milestone-1 Definition of Done (the v1 spine exit, SquaredError)

All of **G0–G3 + G6** green as build-blocking CI jobs, which collectively *are* Green-Spine Alpha on SquaredError:
1. **The five lossless invariant gates green** on every fitted model — Reconstruction / MassConservation / Purity / VarianceSum / ThreeWayEqual (§13.1, Gate G3). Every model `Exact`.
2. **The determinism gate green** — bit-identical serialized `Model` and predictions across `n_threads ∈ {1,2,8}`, on the **quantized** histogram path (§13.4), via integer-associative `Hist` + fixed-`CHUNK_ROWS` reduction + frozen `splitmix64` re-seeding.
3. The standing foundation gates (lints/no-panic/`#![forbid(unsafe_code)]`/`deny(missing_docs)`/doctests/`cargo deny`/MSRV-1.64, §13.8/§13.9) green throughout — they were green from M1.0.

This proves exact ≤3rd-order decomposition end-to-end; **no predictiveness ranking is claimed** (the EBM/TabArena benchmark is the v1.5 release gate, §14.3). Out of this area but adjacent: the full v1 loss set (§05 Phase 4), interaction funnel + monotone + `wht8` (§07 Phase 5), and Python bindings (§12 Phase 7) extend this verified spine; the spine is green before any of them is added.

**Sequencing rationale:** M1.0 freezes the contract and the gates before any algorithm. M1.1–M1.2 are the two leaf dependencies (data + objective). M1.3→M1.4→M1.5 build the engine bottom-up (histogram → split-finder → loop), each gated before the next consumes it. M1.6 makes the model scoreable + serializable (needed for the determinism gate's *bytes* and for ThreeWayEqual's tree-sum side). M1.7 swaps in quantized histograms *after* the full-precision spine is green, per the prompt's explicit "full-precision first, then quantized for the determinism gate" sequencing. M1.8 is the green spine itself (purify + the five checks); M1.9 closes the tree==table equality loop. Nothing builds on an unverified foundation.

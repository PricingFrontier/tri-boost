## Uncertainty & distributional prediction

Point predictions are not enough for pricing. Reserving, capital, large-loss tails, and credibility on thin rating cells all demand a *distribution* (or at least an interval) around the predicted pure premium / claim count. This section inventories every technique in scope that emits prediction intervals or full predictive distributions, judges each against pattern-boost's three invariants, and — critically — asks the question unique to this product: **can a table-decomposed model emit them while keeping the exact ≤3rd-order decomposition?**

The short answer is a spectrum. Some methods (split conformal, quantile/expectile objectives, virtual-ensemble *knowledge* uncertainty on the mean model) preserve the exact tables and are cheap. Others (NGBoost, XGBoostLSS, PGBM, RMSEWithUncertainty) keep the tables exact **per natural parameter** but the deployed uncertainty output is a *nonlinear coupling* of several table sets, so "one exact rating table for the distribution" is not literally true. A few (full SGLB virtual-ensemble *data* uncertainty) need multi-output leaves and a second leaf-update mode that sits in tension with the deterministic Newton core.

### The cheap, exactness-preserving core: conformal + quantile/expectile + virtual-ensemble knowledge uncertainty

These three are the highest effort-to-payoff adoptions because each either changes nothing in the core (conformal, virtual-ensemble knowledge) or only swaps the per-row gradient/Hessian and the leaf-fill rule (quantile/expectile), and all keep the ≤3rd-order tables exact.

**Split conformal (the near-free win).** Split conformal turns *any* fitted point predictor into a prediction interval with finite-sample **marginal** coverage `≥ 1−α` under exchangeability ([Barber et al.](https://ar5iv.labs.arxiv.org/html/2207.12274)). Mechanism: hold out a calibration set, compute conformity scores `s_i = |y_i − F(x_i)|` (or a normalized residual), take the empirical `(1−α)`-quantile `q`, and emit `[F(x) − q, F(x) + q]`. This is one extra pass over a held-out fold and zero changes to the oblivious core or the tables — the point tables are untouched and the interval is a global constant-width band added at predict time. **How to adopt:** ship a thin Python wrapper (or document MAPIE compatibility) over the existing predictor; expose it as `predict_interval()` on the *response* scale, kept entirely out of the table export and the fANOVA/losslessness invariant. **Two caveats that cap the pitch:** (1) coverage is **marginal, not conditional** — for heteroscedastic pricing data (`Var ∝ μ^p`) a constant-width band under-covers some segments and over-covers others, which is exactly the failure actuaries care about; (2) exchangeability breaks under temporal/portfolio drift, so out-of-time intervals degrade. Plain split conformal is therefore a *baseline*, and the adaptive variant below is the real deliverable.

**CQR — the pricing-ideal interval.** [Conformalized Quantile Regression](https://arxiv.org/abs/1905.03222) fixes split conformal's constant-width flaw: fit a lower quantile model `q_lo` (α/2) and an upper `q_hi` (1−α/2), then conformalize the residuals `E_i = max(q_lo(x_i) − y_i, y_i − q_hi(x_i))` and emit `[q_lo(x) − Q_{1−α}(E), q_hi(x) + Q_{1−α}(E)]`. Coverage is guaranteed **and** the width adapts locally — narrow where the model is confident, wide in the sparse high-severity tail. Every component (`q_lo`, `q_hi`) is itself an exact-table oblivious pattern-boost model, so this composes perfectly with the architecture and pairs naturally with the multi-quantile path below. **Honest costs the clean verdict understates:** (a) **multi-table-set deployment** — CQR ships 2 (or 3 with a point model) separate ensembles, so a shop whose *product* is the filed rating table must now file/explain a lower-bound table-set **and** an upper-bound table-set; fANOVA exactness holds per-endpoint, but "one model = one decomposition" no longer holds. (b) Coverage is still **marginal, not conditional** — per-cell calibration needs Mondrian/group-conditional conformal. (c) It needs a held-out calibration split (use cross-conformal/CV+ on small portfolios). Implement either the symmetric variant (single `E_i`) or the asymmetric one (separate quantiles for `E_low`, `E_high`) — both are invariant-equivalent here.

**Quantile / pinball loss (the cheapest distributional capability that touches the loss).** Pinball loss `L_α(y,F) = max(α(y−F), (α−1)(y−F))` estimates the conditional α-quantile; fit several α to build intervals/a discrete distribution. The structure search stays oblivious/depth-3 — **the only change is the leaf-fill rule**: because the true Hessian is 0 almost everywhere, libraries set `h=1` for the split-gain sweep and then **refresh each leaf to the weighted empirical α-quantile of in-leaf residuals** after the structure is grown (the same machinery `reg:absoluteerror`/median needs, and the same the pricing research already earmarks for L1/Quantile v1.5). Each fitted quantile remains a sum of ≤3-feature constant-leaf tables — **exactness is preserved** — but it is a *separate table-set per α*. Five real implementation costs: (1) the leaf refresh needs an inverse row→leaf map plus a weighted-quantile primitive — distinct from the Newton `w=−G/(H+λ)` path; (2) the in-leaf order-statistic refresh is **not histogram-summable**, so it needs a raw-residual pass (the histogram gives exact split boundaries but not the in-leaf median); (3) the gradient-only split signal is coarser than the Newton gain pattern-boost otherwise uses — verify convergence (likely needs more, lower-η trees); (4) **quantiles can cross** (the 0.9 prediction can dip below the 0.5) — XGBoost documents this explicitly; for tail/VaR/capital use, apply post-hoc isotonic-in-α [rearrangement (Chernozhukov)](https://arxiv.org/abs/1905.03222) or fit non-crossing structurally; (5) **monotone constraints don't compose cleanly** — bound-propagation assumes Newton leaf weights, but the post-hoc quantile *overwrites* them, so monotone-in-feature guarantees on the refreshed values need a custom constrained refresh / isotonic post-fit. Implement L1/median as the α=0.5 case of the same code path so one feature delivers both robust severity and intervals.

**Expectile loss (the Newton-clean cousin).** Expectile loss `|α − 1{y<F}|·(y−F)²` is the asymmetric-squared analogue of quantiles, and crucially has a **strictly positive, bounded-away-from-zero Hessian** `h = 2|α − 1{y<F}|` (verified in XGBoost's `regression_obj.cu`, where the weight is a constant asymmetric factor). So it drops straight into the existing Newton split-gain and leaf-weight machinery with no leaf-refresh hack — only the gradient/Hessian formulas change. It is *cheaper to support correctly* than quantile loss and is the natural fit for cost-asymmetric pricing (under-pricing costs more than over-pricing). Two corrections to keep honest: (a) the loss is C¹ but **not** C² — the Hessian is discontinuous at `y=F` — so "smooth positive Hessian" should read "piecewise-constant, strictly-positive Hessian"; (b) the non-crossing guarantee is **not** intrinsic to the loss across separately-trained α — XGBoost engineers it via a reparameterized-gaps single vector-leaf multi-output tree (`multi_strategy=multi_output_tree`, hist-only). Two independent single-α oblivious models can still cross, so pattern-boost does **not** inherit non-crossing for free. Ship the single-α asymmetric-squared objective alongside the v1.5 quantile/L1 work (it's nearly free); defer multi-expectile non-crossing to v2, contingent on a multi-output oblivious tree design.

**Virtual-ensemble *knowledge* uncertainty on the plain mean model (the overlooked freebie).** The virtual-ensemble idea ([Malinin et al.](https://arxiv.org/abs/2006.10562)) is usually bundled with SGLB and RMSEWithUncertainty, but its cheapest, exactness-preserving subset is separable: **predict the existing single-output additive mean model at several `ntree_end` cut points and take the variance over those means** as an epistemic/knowledge-uncertainty signal. This needs *zero* new objective and *zero* leaf-structure change — it fully preserves the exact tables — and could land at v1.5. It is a crude (correlated-members → systematically under-estimated) signal, suitable as an OOD/novelty/under-represented-cell flag rather than a calibrated interval, and that's exactly how to frame it to users.

### NGBoost / RMSEWithUncertainty / LSS — exact *per parameter*, not exact for the distribution

This family is the highest-payoff route to a **full predictive distribution**, and it is the one place where the "exact rating table" promise needs the most careful wording.

**NGBoost.** [NGBoost](https://arxiv.org/abs/1910.03225) predicts a full conditional `P_θ(y|x)` by treating each distribution parameter as its own boosting target. At stage `m` it fits **one base learner per parameter** (e.g. `f_μ`, `f_logσ`) to the *natural gradient* `Ĩ = I_S(θ)⁻¹ ∇S(θ,y)`, where `I_S` is the Fisher information for the scoring rule; the natural gradient corrects the pathological dynamics of optimizing coupled parameters, and a single scalar `ρ` per stage is line-searched. **Two corrections to the naive verdict that are load-bearing for pattern-boost:** (1) the "native oblivious" claim holds *only because the base learner is swappable* — NGBoost's **default** is a depth-3 sklearn `DecisionTreeRegressor`, which is **not oblivious**; pattern-boost must supply its own depth-3 oblivious regressor as the per-parameter learner. (2) The decomposition is exact **per natural parameter** — each `f_μ`, `f_logσ` ensemble is a clean sum of ≤3-feature tables — but the *probabilistic outputs* (quantiles, intervals, tail risk) are a **nonlinear coupling** of the parameter-tables (for a Normal: `μ` and `exp(logσ)` combine through the Gaussian inverse-CDF). So the deployable uncertainty output is **not** a single exact ≤3rd-order table. Market this precisely as *"exact tables per natural parameter, plus a known closed-form link to combine them,"* never as "exact table decomposition of the distribution." Scope v2 to univariate location-scale families (Normal/LogNormal/Gamma/Poisson) — multivariate (Cholesky-parameterized covariance) couples parameters further. Also note CRPS is not implemented for every distribution; LogScore/MLE is the reliable path. Per-feature monotone/interaction constraints must be specified *per parameter*, expanding the API surface.

**CatBoost RMSEWithUncertainty — the existence-proof that oblivious + distributional + table-decomposable is realizable.** This is the single most important reference in the section: a built-in CatBoost loss that trains a **single oblivious-tree model** outputting `(μ, log σ)` per row by minimizing the Normal NLL `0.5·log(2π) + a₁ + 0.5·exp(−2a₁)·(y−a₀)²` with natural gradients (NGBoost-style). It proves the central pattern-boost claim is achievable on symmetric trees. **The exactness story splits by head:** the **mean head `a₀` stays exactly table-decomposable**; the **log-variance head `a₁` is *also* additive and hence exactly tabular** — but only on the **log-σ scale**. Because the user exponentiates `a₁` at the very end, `σ`/`σ²`/intervals are `exp()` of an additive function, which is **non-additive across features** and therefore **not** a clean ≤3rd-order table. The deployable rule: **export the log-variance head as its own exact table set, document that the user exponentiates last, and never fold variance into the mean rating tables** (it is a separate, multiplicative *variance-relativity* table). **How to adopt:** implement multi-output (2-value) oblivious leaves and a Normal-NLL / natural-gradient objective as the first concrete instance of the NGBoost path. **Caveats:** this is a second leaf-update mode (natural gradient ≠ plain Newton), it doubles per-leaf storage, and — critically for pricing — the natural framing of aleatoric uncertainty for Gamma/Tweedie is the **objective's own dispersion/variance**, not a Gaussian RMSEWithUncertainty head, so treat the Gaussian head as a *template*, not a drop-in. Interval calibration on Poisson/Gamma/Tweedie targets is unverified and must be benchmarked before any credibility-grade claim. Depth-3 oblivious is weaker per tree than CatBoost's depth-6 default, so NLL fit quality at our cap is unverified.

**XGBoostLSS / LightGBMLSS (GAMLSS) — the richest pricing target, a superset of NGBoost.** [LSS](https://github.com/StatMixedML/XGBoostLSS) models *all* parameters (location, scale, shape) of a chosen distribution, with **zero-inflated / zero-adjusted families (ZAGamma, ZINB)** as first-class citizens — directly the excess-zero structure of insurance frequency/pure-premium. The reference implementation derives `(g_k, h_k)` per family via PyTorch autodiff, which is **antithetical to a dependency-light Rust core** — pattern-boost would hand-derive gradients/Hessians per family (tractable for Gaussian/Gamma/ZAGamma/ZINB; scope normalizing-flow/mixture families *out*, since flows break the table form). The easily-underestimated part is the **gradient/Hessian stabilization step (MAD/L2)**: naive multi-parameter boosting is numerically unstable because scale/shape gradients differ in magnitude. Same exactness wording as NGBoost: exact **per parameter**, but the deployable artifact is *K additive table-sets plus a closed-form non-additive combiner*, and for some families even `E[Y]` couples parameters. The [GBMLSS/NFBoost benchmark](https://ar5iv.labs.arxiv.org/html/2204.00778) is worth citing as evidence the **parametric-LSS direction beats NGBoost and PGBM** (LightGBM-GBMLSS best average rank 2.5), while NFBoost (Bernstein-polynomial normalizing-flow CDF) is the **non-table-compatible frontier** — flows make the conditional CDF a nonlinear composition of ensembles and so break the ≤3rd-order additivity; keep flows research-only.

### PGBM — mean exact, variance non-additive (the "intervals are not a table" case)

[PGBM](https://arxiv.org/abs/2106.01682) is attractive precisely because it is a **single** standard GBM that yields a predictive distribution at near-point cost, with the family chosen *post-hoc*. Mechanism: per leaf, accumulate the sample mean/variance of the per-sample `g` and `h`, treat the leaf weight `w=−g/(h+λ)` as a random variable with `μ_j ≈ ḡ/(h̄+λ)` and `σ²_j ≈ var_g/(h̄+λ)²`, and propagate prediction mean **and** variance across trees with a constant inter-tree correlation `ρ` (default `≈ log₁₀(n)/100`). At inference, plug `(μ, σ²)` into any location-scale family and read quantiles. **The two-headed verdict is the whole story:** the **mean/point head preserves the exact tables** (bit-identical for MSE; same constant-leaf form but Taylor-shifted values for non-constant-Hessian Poisson/Gamma/Tweedie), but the **variance head breaks exactness** — its recursive `−2αρ·σ^{(k−1)}·σ_j` update is non-additive, couples all trees, and the predictive quantiles are nonlinear in `(μ, σ)`. So PGBM is a legitimate uncertainty add-on **labelled "mean is table-exact, intervals are not."** Two adoption decisions: (1) for the deployed rating table, keep plain Newton leaves `−G/(H+λ)` (which match the CatBoost/XGBoost point predictor) and compute PGBM moments as an **optional parallel head**, *not* the Taylor-corrected `μ_j` (which would make the exported table differ from the standard predictor); (2) PGBM is a location-scale *regression* method — it does not natively emit a Poisson/Gamma/Tweedie predictive law, so the `(μ,σ)→family` mapping under a log link is a modeling assumption needing care (natural-parameter vs response-scale moments). The single global `ρ` is a crude calibration knob to tune/validate. The leaf five-moment accumulation is cheap — it extends the existing g/h histogram pass.

### SGLB virtual ensembles — the only route to *epistemic* uncertainty from one model

[SGLB](https://arxiv.org/abs/2001.07248) injects Langevin noise into the boosting gradient step (with `model_shrink_rate=1/(2N)`, `diffusion_temperature=N` under `posterior_sampling=True`) so the trained trajectory samples a Gibbs posterior over models; **truncations/checkpoints of the single model then form a "virtual ensemble"** whose prediction spread estimates *epistemic / knowledge* uncertainty — flagging rows far from training support (novelty/leakage detection, valuable for pricing). **Decompose the proposal into two separable mechanisms, because they have opposite invariant profiles:** (A) **SGLB/Langevin *training*** is native to oblivious trees and leaves the point predictor exactly table-decomposable — it is arguably a standalone robustness/convergence knob (global convergence on multimodal losses) worth its own research record, independent of uncertainty; don't let "breaks exactness" scare us off it. (B) the **virtual-ensemble uncertainty readout** is the only part producing a non-additive output, and even then the *mean* tables stay exact while the epistemic scalar is a non-table diagnostic — acceptable as an auxiliary output. **The pricing-objective gap is the binding constraint:** CatBoost's *data* uncertainty needs RMSEWithUncertainty (Gaussian NLL); there is **no Poisson/Gamma/Tweedie-with-uncertainty loss** in CatBoost. So for pricing we could realistically ship only **knowledge** uncertainty (truncation spread, loss-agnostic) unless we derive our own Tweedie dispersion head (novel, research-tier). Engineering cost is non-trivial: Langevin update + shrink/temperature plumbing, model-truncation readout, and — for any *data* uncertainty — multi-output (2-dim leaf) oblivious trees plus a table exporter that no longer assumes scalar leaves. CatBoost docs confirm virtual-ensemble uncertainties are **systematically under-estimated** (correlated members) — frame as an OOD flag, not a calibrated interval. This is the heavier v2 piece; the cheap subset (knowledge uncertainty on the plain mean model, above) gets most of the epistemic value at v1.5.

### IBUG — non-Gaussian local intervals that reuse the trained model

[IBUG](https://arxiv.org/abs/2205.11412) extends any trained GBRT point predictor into a probabilistic one using the **tree-ensemble kernel**: `affinity(x_i, x_test) =` the number of trees where `x_i` lands in the same leaf as `x_test`; the `k` highest-affinity training instances form a local empirical distribution (mean from the GBRT, higher moments / arbitrary parametric fit from the neighbors). It is **distinctly synergistic with oblivious trees** — the 3-bit leaf code makes the leaf-co-occurrence kernel trivial and fast — and it **reuses the trained model with no retraining**, giving flexible *non-Gaussian* local intervals matching or beating NGBoost/PGBM on 22 datasets. The fANOVA tables are untouched (it is purely a post-hoc, instance-based readout). The one deployment cost is that it needs **training-set retention at inference**, which conflicts with the "model is just a small set of lookup tables" deployment story — so it is an opt-in scoring mode, not a table. Strong differentiated v2 add-on once the core ships.

### Venn-ABERS — calibration-with-uncertainty for classification objectives

[Venn-ABERS](https://arxiv.org/pdf/1211.0025) is a conformal-flavored calibrator for **binary classification** (lapse / conversion / large-loss-propensity / fraud-flag) that outputs a *pair* `(p₀, p₁)` bracketing the true calibrated probability, with a distribution-free validity guarantee under exchangeability: fit two isotonic regressions of the calibration labels on the score (one assuming the test label is 0, one assuming 1); the point estimate is `p₁/(1−p₀+p₁)`, and the interval width flags low-confidence (sparse-score) regions — a governance asset. As two isotonic warps of the single score, it sits exactly where the link function sits and leaves the score tables untouched (**none**) — *provided* it is kept as a separate score→probability layer and the tables are exported on the score scale. If instead the *calibrated probability* is the deployed/explained artifact, then like any nonlinear warp of `F` it breaks the additive decomposition (this non-additivity already exists for the uncalibrated logit-link model, so it cannot be charged to Venn-ABERS specifically). **Two scope limits:** it is binary-only (the regression extension pattern-boost would need for Poisson/Gamma/Tweedie is research-grade and largely duplicates the planned autocalibration step), and it is model-agnostic — "native" here means "orthogonal," not that it exploits the depth-3 structure. v2; secondary to the regression distributional path that pricing actually leads with.

### What breaks the table form, and the one verdict-flip to remember

Three lines in this scope genuinely change the table form or break exactness, and they must never silently contaminate the lossless pipeline:

- **Variance / log-variance heads** (RMSEWithUncertainty, NGBoost scale parameter, PGBM variance channel): exact in the *log* parameter, **non-additive after `exp()`** → ship as a separate multiplicative variance-relativity table, never folded into the mean rating tables.
- **NFBoost / normalizing-flow distributional output** and **Distributional Random Forests (MMD-split)**: the predictive CDF is a nonlinear composition of ensembles / a weighted-empirical distribution from a non-oblivious forest → **breaks ≤3rd-order additivity**; both are research/benchmark references only (DRF is incompatible on all three invariants and is a conceptual contrast, not an implementation target).
- **Multi-quantile / multi-α everything** (quantile, CQR, multi-expectile): each α/endpoint preserves exactness *individually*, but the deliverable becomes **K rating-table stacks**, not one — a deliberate UX/governance decision (per-quantile tables vs a single point table + interval tables), not a free axis.

The **verdict-flip to remember**: NGBoost / LSS / RMSEWithUncertainty / PGBM all preserve exactness **per parameter/head**, so the right framing is *"a stack of exact table sets plus a known closed-form (usually nonlinear) combiner,"* not *"an exact table decomposition of the predictive distribution."* The only members of this section that are **uncertainty-as-an-exact-table** are virtual-ensemble knowledge uncertainty on the plain mean model and the per-α quantile/expectile table-sets; everything richer trades the single-table promise for distributional richness — which is the correct trade for reserving and tail-risk, but must be stated, not assumed.

Finally, two cross-cutting reminders for whoever specs this: every interval/distribution method here lives on the **response scale** and should be exposed via a separate `predict_interval()` / `predict_dist()` surface kept *out* of the fANOVA losslessness invariant; and because deviance/NLL minimization does **not** guarantee total balance ([Denuit et al. 2021](https://arxiv.org/abs/2103.03635), already flagged in the pricing research), any credibility-grade interval claim must be paired with the A/E balance and autocalibration checks before it is sold as actuarially defensible.

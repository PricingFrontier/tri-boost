## Interpretability, interaction-selection & calibration

This section covers techniques that explain the trained model, *select which interaction tables to keep*, and *calibrate* the output. The recurring theme is that tri-boost's exact ≤3rd-order table decomposition turns most of these from approximate, costly add-ons into **closed-form table reads** — but a sharp line separates (a) attributions/effects that are exact functions of the purified tables (free, both invariants intact), (b) interaction-selection that *sculpts* which tables exist (train-time, exactness preserved), and (c) calibration warps that act on the *aggregate score* and therefore **break the additive table decomposition** unless handled with care.

### The big win: exact attributions and effects are *free table reads*, not model probes

Every standard "explanation" artifact for tree ensembles — TreeSHAP, SHAP interaction values, PDP, ICE, the H-statistic, Sobol importances — is, for an additive ≤3rd-order model, a **closed-form function of the purified tables**. tri-boost already materializes `f0 + Σ_i f_i + Σ_{ij} f_ij + Σ_{ijk} f_ijk` exactly, so it should *never* run the costly tree-traversal algorithms. The unifying result is [Bordt & von Luxburg (2023)](https://arxiv.org/abs/2209.04012): for any function given as an additive decomposition, the marginal Shapley value is

> φᵢ(x) = Σ_{u : i ∈ u} f_u(x_u) / |u|

i.e. each interaction component's value is split *equally* among its members. For tri-boost (max order 3) this is at most `(#mains + #pairs/2 + #triples/3)` table lookups per row — exact, not Monte-Carlo, and bit-consistent with the deployed rating tables.

**The load-bearing caveat is the reference measure.** The equal-split identity gives **interventional/marginal SHAP** under a *product-of-marginals* baseline and **observational/conditional SHAP** under the *joint-conditional* baseline. tri-boost's default purification measure is the Laplace-smoothed empirical *product* of marginals, so the closed form yields **interventional SHAP**, which differs materially from conditional TreeSHAP when pricing features are correlated (it attributes mass to off-manifold combinations — "true to the model," which is arguably what you want for an auditable rating table). **The API must label this "interventional SHAP," never just "SHAP,"** or it will silently disagree with the conditional TreeSHAP users compare against. A second subtlety: under the *joint/weighted* (Hooker) measure the naive equal-split is **not** guaranteed exact (components are only hierarchically orthogonal, not the standard conditional functional decomposition), so the closed-form SHAP surface should be restricted to the product-measure mode, or the correct aggregation re-derived for the joint case first. This is why path-dependent and interventional **TreeSHAP, Fast TreeSHAP, and GPUTreeShap are all `skip`**: they optimize a computation tri-boost obtains analytically in O(1) per row, and path-dependent TreeSHAP additionally returns an *approximation* (path cover ≠ true conditional). The only legitimate use of stock SHAP here is as an **independent oracle in tests** — TreeSHAP-vs-table-attribution agreement is an excellent end-to-end correctness invariant for the lossless-decomposition pipeline.

**PDP and ICE collapse onto the tables identically.** Under the product reference measure, `PD_i(x_i) = f0 + f_i(x_i)` and `PD_ij(x_i,x_j) = f0 + f_i + f_j + f_ij` — the partial-dependence curve *is* the purified main/pairwise effect (plus lower-order terms it contains). Ship "PDP/ICE" as a thin view over the tables so users see a familiar artifact with zero new math. Two precise notes: (1) the PD = table-sum identity holds only under the **unsmoothed** product measure, *approximately* under the Laplace-smoothed default, and **not at all** under the joint measure — the view must display which measure it assumes; (2) sklearn's `recursion` method is cover-weighted traversal (conditional PDP), **not** the product-measure PDP, so do not cite it as the mechanism. ICE is `ICE_x(z) = f0 + f_i(z) + Σ_j f_ij(z, x_j) + Σ_{j<k} f_ijk(z, x_j, x_k)` — the row's own slice; parallel (non-fanning) ICE curves ⟺ feature `i` has no interactions, which is directly checkable from whether any `f_ij`/`f_ijk` containing `i` is nonzero. Both should sweep over the merged-threshold grid Ωᵢ so curves are exact piecewise-constant steps with zero approximation error.

**ALE is the one effect-plot that genuinely *differs* and is `skip`.** Accumulated Local Effects conditions on local neighborhoods rather than marginals, avoiding the product-measure extrapolation that PDP suffers under correlation. But for tri-boost this is the *same concern* the reference-measure choice already addresses: ALE-style faithfulness comes from the joint/weighted `w` in purification (next subsection), so ALE is at best a redundant cross-check and at worst a competing artifact whose tables are **not** the deployable lossless tables. Treat it as a research note ("does centered ALE under joint `w` coincide with our joint-`w` purified tables on piecewise-constant ensembles?"), not a feature.

### Interaction selection: sculpting *which* ≤3-feature tables exist (the v1.5 core)

The order-3 cap comes free from depth-3 obliviousness — but an *unconstrained* order-3 oblivious GBM can emit a combinatorial pile of 3D tables, defeating the "rating-table-you-can-read" promise. The entire value of this cluster is **bounding and ranking the table set**, and all of it is train-time or post-hoc selection that **preserves exactness** (it restricts which tables are nonzero, never their order or form). There are two complementary families:

**Ranking by exact variance (post-hoc, free).** The **H-statistic** numerator `PD_jk − PD_j − PD_k` is *precisely* the centered pairwise component `f_ij`, so `H²_jk = σ²(f_ij)/σ²(PD_jk)` is an exact variance ratio with **zero model calls** — fixing the canonical complaints about H (O(n²) evaluations, Monte-Carlo instability). Even better, **Sobol indices** `S_u = σ²(f_u)/σ²(F)` and **Shapley effects** come straight from the purified table variances and are the *primary* importance/selection primitive. Three precise guardrails: (1) both are exact-from-tables **only under the product measure** (which is the default, so fine) — under the joint measure `σ²(F) ≠ Σ_u σ²(f_u)`, so to keep "importances summing to 1" you must use **Shapley effects** (which satisfy efficiency by construction), not raw Sobol ratios; (2) H has intrinsic pathologies (can exceed 1; spuriously large when both main effects are weak; denominator blow-up) that survive exact computation and need clamping/warnings; (3) keep the Sobol/Shapley-effects variance fraction as the *canonical* importance and H as a familiar secondary view.

**Filtering at train time (sculpts the model).** **FAST** ([Lou et al. 2013](https://www.cs.cornell.edu/~yinlou/papers/lou-kdd13.pdf)) ranks candidate pairs by the RSS reduction of a 4-quadrant residual fit using cumulative histograms in O(b²) per pair — and its histogram-DP idea maps perfectly onto tri-boost's binned engine. **Heredity/strong-hierarchy** ([GAMI-Net](https://arxiv.org/pdf/2003.07132)) admits an interaction only if its parent effects are already important. Used together, these feed the **interaction-constraint** machinery: allow a tree to split a triple only if its sub-pairs cleared the screen, keeping the realized ≤3-feature support — and thus the exported table set — small *without scanning C(n,3)*. Three caveats: (a) FAST is *literally* a pairwise method; the triple extension is real but candidate count is cubic — prefer **heredity/composition** ("admit a triple only if its sub-pairs are admitted") over a raw `O(b³)` triple-FAST scan; (b) FAST's RSS-on-residual objective differs from the booster's Newton summed-gain, so use it as a **soft pre-filter** feeding constraints, with final selection by **exact purified Sobol variance after training**, not as a hard gate that can amputate interactions the GBM would have found; (c) the [GAMI-Tree](https://arxiv.org/pdf/2207.06950) lesson is that a *naive two-stage* (fit all mains to convergence, then fit interactions on the residual) can mis-converge under correlation — so prefer **joint boosting over the realized supports followed by a single purification pass** rather than EBM-style staging. Note this is FAST/heredity is largely already-committed scope; the net-new content is using them as a *train-time admission* filter (not just an explain-time pruner) and the triple extension.

Two adjacent items round this out at low cost: **bagging** ([InterpretML inner/outer bags](https://interpret.ml/docs/python/api/ExplainableBoostingRegressor.html)) averages multiple oblivious ensembles' tables to yield smoother tables **plus per-cell standard-error bands** — a genuine credibility feature — but requires projecting every bag's tables onto a **common merged-threshold union grid** (tri-boost picks thresholds per-tree, so bags disagree on cuts *and* triples; the averaged set is the union, denser), and the SEs are an annotation, *not* a fANOVA component (compute spread *after* per-bag purification). And the **variable-interaction-network** graph is a pure visualization layer over the exact H/variance values — cheap, high-value for actuarial review, no core impact.

### The reference-measure decision: the one purification mode that changes table *content*

This is the most consequential design call in the section, and it is mis-scored as `breaks/changes_table_form` in the source list — it is **`preserves_exactness`**. Default Lengerich purification uses a *product-of-marginals* measure; under correlated pricing covariates this evaluates components at low-density feature combinations (the Hooker/Owen extrapolation artifact). Switching the reference measure `w` to the **joint empirical density** (Hooker's weak/hierarchical orthogonality) makes the tables faithful to the real data distribution — the same Lengerich mass-moving algorithm, just with a non-product `w`, converging geometrically in a few passes. Crucially: **leaves stay piecewise-constant, components stay on their ≤3 features, and the prediction sum is conserved bit-for-bit.** Only table *values* change, not form or order. Expose `w ∈ {uniform, empirical-product+Laplace, joint/empirical}` as a config flag on the *existing* purification step; because purification is post-hoc and cheap, let users **recompute tables under a different `w` without retraining**, and always surface which `w` produced the displayed tables (Lengerich shows the choice can change purified effects dramatically). The genuinely open research question is not the algorithm but **estimating the joint density in 3D**: a piecewise-constant empirical `w` on the merged 3D grid has many empty cells and needs Laplace smoothing or a factorized/copula approximation to stay well-conditioned and reproducible. This mode is the principled answer that ALE only gestures at, and it pairs naturally with Sobol/Shapley-effects importances and SHAP (recomputed under the matching `w`).

### Calibration: the exactness fault line (score-space vs response-space)

Calibration is where most of these techniques **break the additive table decomposition**, and the distinction must be enforced architecturally. The rule: **any monotone/nonlinear 1-D warp `g` of the aggregate score, applied *after* the table sum, makes `g(F(x))` non-additive in the per-feature tables** — `g(Σ f_u) ≠ Σ g(f_u)`. So calibration must be one of two things: (i) an affine/intercept adjustment that folds back into the tables exactly, or (ii) a *separately-declared, deployment-time calibration curve* that sits where the link already sits and is **never re-decomposed** as if it were part of the rating tables.

**The exactness-safe core (v1).** **Global mean/intercept re-anchoring** is the cheap, always-correct fix: deviance-trained GBMs lose the GLM canonical-link balance property (`Σμ̂ = Σy`), so re-solve a single scalar `δ` and fold it into `f0`. For log-link Poisson with offset, `δ = log(Σ w·y / Σ w·e·exp(F))` in closed form (use the *same* exposure/weights as training; non-log links need a trivial 1-D root-find). This shifts only the constant term, so it is **fully exactness-preserving** — re-assert the losslessness test afterward (only `T_empty` changed). Default-on. The **reliability-diagram / A-vs-E balance report** is reporting-only and `v1`: bin by predicted rate (exposure-weighted), show global and per-decile A/E, Lorenz/concentration curves, and the [Denuit (2021)](https://arxiv.org/abs/2103.03635) autocalibration check — and exploit tri-boost's unique edge by running balance diagnostics **per table** (per main effect, per interaction cell), a real differentiator over black-box GBMs.

**The exactness-*breaking* calibrators (declare separately).** Full **autocalibration / local balance correction** ([Denuit, Charpentier & Trufin 2021](https://arxiv.org/abs/2103.03635)) regresses `Y` on the model's own scalar prediction `π̂(X)` to get `π_BC = E[Y | π̂(X)]`, restoring *local* balance and improving lift. The catch: this is a generally-nonlinear 1-D warp of the aggregate score, so the deployed `recalib(Σ tables)` is **no longer equal to the sum of lookup tables** — only the trivial *global affine* sub-case folds in cleanly; the novel *local* correction is precisely the part that breaks exactness, and isotonic/monotone constraints do not rescue it. Ship it as a `fit(autocalibrate=True)` **post-step with a loud caveat**, keep the exact tables for `π̂`, and never fold the local curve into the table export. **Multicalibration/multibalance** ([Hébert-Johnson 2018](https://proceedings.mlr.press/v80/hebert-johnson18a/hebert-johnson18a.pdf)) is the same family extended to per-protected-subgroup balance (`v2`, strong regulatory value), with the identical exactness-break on the deployed predictor and the caveat that the published method handles essentially one categorical sensitive attribute. **Tweedie-dominance / convex-order model selection** ([Denuit 2019](https://ideas.repec.org/a/eee/insuma/v89y2019icp128-139.html)) is the *safe* companion: the Lorenz/concentration-curve and ordered-Gini comparison is **evaluation-only** and touches nothing — but note its "dominance for all ξ≥1" guarantee is *conditional on autocalibration*, so if you deploy the autocalibrated predictor to earn that guarantee, you again leave the exact-table regime.

**Classification calibrators** (relevant only for the Logistic objective) split cleanly by scale. **Isotonic regression** (`v1.5`) is the highest-value: a PAV monotone fit on a held-out set, and — the key insight — isotonic regression of `y` on the score is **exactly the monotone realization of the balance-correction step**, so *one PAV implementation serves both classification calibration and regression autocalibration*. **Platt/sigmoid** (`v2`, `p = σ(A·F + B)`) and **temperature scaling** (`v2`, single scalar) are affine logit warps that, on tri-boost's **logit/relativity scale**, can be **folded into the tables exactly** (multiply every leaf by `A`, add `B` to `f0`) — so on the score scale these *preserve* exactness; only the post-link probability surface is non-additive, and that non-additivity already exists for the uncalibrated logit-link model. **Spline/I-spline** (`v2`) gives a smooth monotone score warp (prefer the **monotone I-spline** variant — plain SplineCalib is *unconstrained* and can be non-monotone, contradicting the monotone rating tables). **Beta calibration** (`research`) and **Venn-ABERS** (`research`) act on the **response/probability scale** and so break exactness for the deployed number; Venn-ABERS' distribution-free validity interval is an attractive governance feature but is classification-only and largely duplicates the planned autocalibration for regression. The integration rule across all of them: apply the warp **outside** the table decomposition as a documented score→prediction map (exactly like the inverse link), and keep exporting tables on the raw-score scale — folding calibration into leaf values pollutes the decomposition.

### What's external validation, not adoption

Several entries are best treated as **prior-art that validates tri-boost's thesis** rather than code to import. [NODE-GAM](https://arxiv.org/pdf/2106.01613), [GAMI-Tree](https://arxiv.org/pdf/2207.06950), and the GA2M/EBM lineage all confirm that "oblivious tree = lookup table = explanation" and that the **screen→purify→monotone-tables** pipeline works inside boosting — but they all **cap at pairwise (order 2)**. tri-boost's exact *3rd-order* extension via hard oblivious trees is the differentiator. NODE-GAM is `incompatible` because its trees are *soft/differentiable* (a false-friend "oblivious") and it breaks exactness; GAMI-Tree uses linear-leaf model-based trees (changes table form). EBM's **post-hoc `monotonize()`** (isotonic on a single 1-D term) edits tables away from the ensemble and so forfeits losslessness — adopt EBM's *in-fit* monotone/smoothing/missing handling (which preserve exactness) but treat any post-hoc table-editing as opt-in deployment processing tracked **separately** from the bit-exact artifact. **Global surrogacy** inverts cleanly into a marketing angle: tri-boost is *itself* the exact glass-box, so it can serve as a **high-fidelity order-3 surrogate FOR someone else's black box** (fit to `(X, blackbox(X))`, read the audited tables) — a use-case note, with the standard `R²<1` fidelity caveat, never something to build into the core. **NID** ([Tsang 2018](https://arxiv.org/pdf/1705.04977)) / the actuarial NN-interaction-detection variant (correctly attributed to **Havrylenko & Heger 2023**, *not* Richman & Wüthrich) is a `research` pre-screen that produces an a-priori interaction allow-list — use only as a **soft prior** widening the constraint set, never a hard exclusion gate, since it detects interactions in the *trained NN*, not ground truth.
